Undo Changes command (Edit menu)

 

This command undoes the last action if it was an action that modified the file.  If the last action was not a file modification command then all actions up to, but not including, the last file modification are undone.  Using the Undo Changes command can be a lot faster than undoing every single action using the Undo command, particularly if there are a lot of display changes and you just want to undo the last change.

 

Remember that HexEdit keeps track of display changes for each window open on a file.  If you undo a file modification in one window the file modification change is reflected in all other windows open on the file.  This means that all display changes in the other windows have to be undone back to the point of the file change, in order to keep the display consistent with the file contents.

 

If you try to undo a file change that was made in a different window, HexEdit will ask if you really want to do so.
