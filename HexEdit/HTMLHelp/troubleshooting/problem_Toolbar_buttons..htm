Toolbar buttons are wrong

 

If some HexEdit registry settings are incorrect the pictures shown on the toolbar buttons may not match the action of the button.  (This happened to some users who had customized the built-in toolbars when they upgraded from version 2.0 to 2.1.)

 

To fix the toolbar(s) use the Customize command and go to the Toolbars page.  Select the built-in toolbar that has a problem and click the "Reset" button, or just click the "Reset All" button to reset all built-in toolbars.  Note that any customization of the toolbars you have made will be lost.

 

Starting in version 2.1 toolbars (and menus) have become intelligent about new releases.  Changes to built-in toolbars are merged with customizations when upgrading.  However, a wise strategy is not to customize the built-in toolbars but create your own toolbar(s) -- thence upgrading to a new version should never cause problems.
