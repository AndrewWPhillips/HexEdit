A search unexpectedly failed

 

There could be many reasons that your search failed.  It is very common to be using the wrong search options.

 

Here are some things to try:

  • double check the bytes that you are searching for

  • if searching to start/end of file try changing the Scope to "Whole file" and repeating the search

  • rather than typing in values to find the next occurrence in the file, it is less error-prone to select the bytes and press Ctrl+F3

  • double check the type of search (eg. you might have intended to do a hex search but instead searched for ASCII text)

  • for a hex search make sure that the alignment is correct (or use Byte alignment to disable any alignment checking)

  • if using hex in uppercase make sure you haven't mistaken an 8 for a B or a 0 for a D

  • if using hex in lowercase make sure you haven't mistaken a 6 for a b

  • for a number search make sure that the format and byte order (little- or big-endian) are correct

  • for a text search turn off "whole word" and try again

  • try a case-insensitive search since it is easy to get the case of characters wrong (and it often turns up other useful info)

  • when searching for text make sure you haven't mistaken a lowercase-L for a 1, an O for a zero etc

  • if searching for text in a .DLL or .EXE file try a Unicode search as well as an ASCII search
