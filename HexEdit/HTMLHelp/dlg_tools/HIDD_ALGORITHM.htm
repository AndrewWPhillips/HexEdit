Encryption Algorithm dialog

 

 

Use this dialog to select the algorithm used with the Encrypt and Decrypt commands.  When decrypting you must use the same algorithm with which you encrypted.

 

HexEdit provides an internal algorithm plus access to all Crypto API algorithms installed.  For further details see Encryption.

 

Algorithm

Use the drop down list to select an encryption algorithm.  The HexEdit Internal Algorithm is always available at the top of the list.  If the Crypto API is installed (which includes the Microsoft Base Cryptographic Provider v1.0) then at least another two algorithms will be present: RC4, a stream cipher, and RC2 a block cipher with a block length of 8 bytes.

Block Length

This indicates whether the currently selected algorithm is a block or stream cipher.  For block ciphers it shows the block length.

Key Length

This indicates the key length of the currently selected algorithm.  The larger the key length the better the encryption.  Traditionally algorithms used a key length of 40 to 64 bits.  Nowadays, using special hardware these would be susceptible to brute force attacks, so new algorithms use a 128 bit or more key length.
