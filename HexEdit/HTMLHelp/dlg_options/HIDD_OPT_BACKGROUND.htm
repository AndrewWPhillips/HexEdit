Background Processing Options Page

 

 

This page allows you to change global options relating to file and editing commands.

 

Background: Searching

When the current search text is changed (either in the Find Tool or the Find dialog) a background search is started for all other occurrences of the same text. The search is done using low-priority background thread(s), so that you should not notice any difference to your normal use of HexEdit. (The search is done using a background thread with PRIORITY_LOWEST in the active file and even lower-priority (PRIORITY_IDLE) threads for any other open files.)

 

When the background search is finished the found occurrences are highlighted with a light-green background. Also when the background search is finished subsequent searches for the same search text are much faster – this is particularly noticeable for a large file.  While the search is in progress a progress bar is shown in the Status Bar and a count of the number of occurrences is shown when it is complete.

 

You would normally leave this option on unless you had reason not to. You might turn it off if you do not need the facility and you are working with large files on a network and the file access is slowing down the network. Or perhaps you have another background task that is CPU intensive, so you want to give CPU time to that instead of the background searches.

 

Background: Statistics

When this option is selected a low-priority thread is used to scan all the bytes of the file and analyze them.  The results are shown in the graph in Counts (Statistics) Page, and the CRC and message digest values in Checksum Page.

 

Exclude the from background processing

You may want to exclude some types of files from background processing.  This is usually done for performance reasons - for example performing background processing on a large network file may result in excessive network traffic.
