Workspace Editing Options Page

 

 

This page allows you to change global options relating to file and editing commands.

 

Document Editing Options >>

This button jumps to the Document Edit Options page for options related to editing of the active file.  This button is disabled if there is no file open.

Undo

Number of consecutive changes to merge

This merges consecutive related changes (inserted, deleted or modified bytes) so they can be all undone together  For example, if you backspace to delete many bytes it is tedious to undo each deletion one byte at a time. Using a large value means you can quickly undo substantial changes.

 

Intelligent undo

This removes redundant cursor movements from the undo stack.  For example, if you move the cursor right a few times then reverse the action by moving left, these movements are not saved and you do not have to tediously step back through them when you want to undo some earlier action.

Clipboard

These options allow you to control how HexEdit works with the clipboard.  Note that for all options all binary data (including nul bytes) will be copied and pasted correctly within the application.

 

Also note that Windows cannot handle very large amounts of data on the clipboard.  This depends on the amount of memory is the system but is in the region of 20 to 50 MBytes.  In this case HexEdit will use it's own internal clipboard format instead of the formats discussed below in order to handle very large clipboard operations.  This internal format relies on creating temporary files to store the actual data.

 

Use binary data/characters

This is the way that HexEdit 1.0 to HexEdit 3.6 stored binary data on the clipboard. It is actually added to the clipboard in two different formats:

1. Binary data format compatible with Visual Studio hex editor - this allows copying and pasting of all binary data (including nul bytes) and compatibility with Visual Studio

2. Text (converted as appropriate - eg if in EBCDIC mode) - this allows pasting text into text editors etc but loses nul bytes

Use Hex Text

This option has recently been requested by several users.  The data is stored as text with each byte converted to two hex digits with a space between each byte.  This allows the data to be pasted into a text editor as hex text but can also be pasted back into HexEdit with no loss of information.

Automatic

When this option is selected the data to be placed on the clipboard is analyzed and if text the first of the above options is used (binary data + characters) -- this allows the data to be pasted into a text editor as the actual text (using the current character set).  If the data contains non-text bytes then the second option above is used (hex text) -- this allows pasting the data into a text editor as hex text.

Use hex text in hex area, binary data/characters in text area

When this option is selected the data to be placed on the clipboard depends on whether the selection was made in the hex area or the text area.  If the selection was made in the text area the first of the above options is used (binary data + characters) -- this allows the data to be pasted into a text editor as the actual text (using the current character set).  If the selection was made in the hex area then the second option above is used (hex text) -- this allows pasting the data into a text editor as hex text..

Scroll/Zoom

Allow scroll past ends

Dragging the vertical scroll bar to the bottom moves the file display to the end, with the end of the file at the bottom of the window.  Using this option you can click the arrow at the bottom of the scroll bar (or use the mouse wheel) to scroll into the blank area past the end of file. I am not sure why, but many programs allow this so it is provided as an option in HexEdit.  (The horizontal scrollbar works similarly.)

 

Autoscroll acceleration

When selecting with the mouse and you drag past the top or bottom of the display the window will "autoscroll" to allow you to select more of the file than can be displayed at once.  As you drag the mouse further from the edge of the window the autoscroll accelerates so that you can select large blocks easily.  This slider allows you to set how quickly the autoscroll accelerates.  Dragging all the way to the left disables autoscroll acceleration, so the autoscroll speed is always the same.

 

Invert Ctrl+mouse zoom

HexEdit has always had the ability to zoom the display (ie, change the Font size) by using the mouse wheel while holding down the Ctrl key.  More recently a convention has emerged which zooms with the Ctrl+wheel but in the opposite direction.  This checkbox means you have the choice to configure HexEdit Pro either way.  Note that this setting also affects the Aerial View.
