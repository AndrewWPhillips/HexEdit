Searching

 

Quick searches can be performed with the Find Tool on the Edit Bar. The Find Dialog provides similar facilities but more options. Searches are done with the Boyer-Moore algorithm which, apart from being very fast, has the peculiarity that the longer the string of (unique) characters you are searching for the faster the search, although the limiting factor is likely to be the speed of your disk or network.

 

Find Tool

 

The Find Tool on the Edit Bar allows you to search for bytes by either entering them as hex digits, or as a string. String searches can be case-sensitive or case-insensitive, and use EBCDIC rather than ASCII characters if you are currently viewing characters using the EBCDIC character set. You can also quickly start a case-insensitive, case-sensitive or hex search using the F4, F5 or F6 key.

 

The Find Tool also retains a history of the most recent searches performed. This history list is retained between invocations of HexEdit and is shared with the pages of the Find dialog (see below).

 

Find Dialog

 

The Find dialog provides a few more options than the Find Tool. For example, you can use wildcards, and search for whole words, perform a search and replace operation etc.  The only disadvantage of the Find Dialog is that it takes up more screen space than the Find Tool, but it does automatically get out of the way of the cursor if it can.

 

The find dialog is divided into several pages.  Every page has the same direction (up/down) and scope controls.  The scope is typically "to end of file" (or "to start of file" if the direction is up) but you can search the whole file and even all open files.  After a successful whole file search is done, the scope automatically switches to "to end of file" mode so that a subsequent search does not find exactly the same thing.  The "to mark" scope is useful if you just want to search through part of a large file.

 

The "Basic" page allows you to quickly specify a search item as a string or in hex.  If the characters you enter look like hex bytes (ie contain only an even number of hex digits, with optional spaces) then a hex search is done, else the characters are treated as a string to search for.  A string search has the option of being whole word only and case-insensitive.  The string search is done in EBCDIC if the active file is in EBCDIC display mode else ASCII is used.

 

The "Hex" search page is useful if you want to ensure you perform a hex (not text) search.  It also allows you to specify a bit mask to remove certain bits from the search.  There is also an option to specify the alignment so you could for example only search for a number when it occurs on a word boundary.  As you enter the bytes in hex the corresponding characters are filled in in the text page (see below) in the selected character set, which allows you to check ASCII and EBCDIC values and even convert between character sets.

 

The "Text" search page is useful if you want to ensure you perform a text (not hex) search.  You can also specify a wildcard character that matches any character.  As well as the whole word and case-insensitive search option you can also explicitly specify an ASCII (actually ANSI), EBCDIC or Unicode character set search.  As you enter text the corresponding hex values are entered in the hex page (using the current character set).  Hence you can flip between page to combine options, eg, if you wanted to perform a text search but with alignment restrictions.

 

The "Number" page allows you to search for number in a variety of integer and floating points formats and sizes, and in both big- and little-endian byte order.  Like the hex page there is also an option to specify the alignment of the number found.

 

The "Replace" page allows you to perform a search and replace operation.  Like the simple page you can use a text or a hex search and replace -- hex is only used if both the search and replace items appear to be hex.  The "Replace" button allows you to interactively find and replace items one at a time, whereas the "Replace All" button allows you to replace all occurrences within the current scope.

 

Other Find Commands

 

Other “find” commands allow you to repeat a search, using the current search bytes, in either direction. Like many Windows programs (like NotePad) the find next command is mapped to F3 (in the default keymap) and find backwards is mapped to Shift+F3.

 

The “Find Selection” command allows you to search for the next occurrence of the currently selected bytes.  This is mapped (as in Visual Studio and other programs) to Ctrl+F3.
