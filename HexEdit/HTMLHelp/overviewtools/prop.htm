Properties Dialog

 

The Properties dialog lets you easily view the properties of the file and the properties of the byte(s) at the cursor in the active window.  There are six pages:

 

Page

Purpose of Fields

Summary

You can enter and view comment fields (category, keywords, comments) here.

You can also see how long the file has been edited or viewed.

This page also shows the size of the file on disk and in memory (which may be different if bytes have been deleted or added).

File

Shows information about the current file.  None of these values can be modified.

All fields are as for the file on disk except the size on disk which is shown in the Summary page..

Byte

Shows the value at the cursor in hex, decimal, octal and binary.

You can edit the value as decimal, octal or binary (use the hex area for hex editing).

Double-click any of the eight bits in the Binary field to toggle it between 0 and 1.

Also shows the byte as ASCII and EBCDIC and the 2 bytes after the cursor as Unicode.

Decimal

Shows the 8, 16, 32 and 64 bit value after the cursor in decimal.

The unsigned and 2's complement formats are the most commonly used.

Big-endian byte order is used for Motorola based systems and many UNIX systems.

All values can be modified according to the current format and endianness.

IEEE FP

Shows IEEE or IBM floating points formats and C# Decimal.  The mantissa and (signed) exponent are also shown.

The value can be edited, but the mantissa & exponent cannot be changed separately.

The IEEE format is by far the most commonly used floating point format particularly on microcomputers.

Date

Shows dates/times in several (selectable) common formats.

You can change the dates, set them to the current date and time etc.

 

The Properties dialog is modeless, so you can leave it displayed as you work with a file; it will even try to get out of the way if you move the cursor "under" it.  As you move around or swap between files the properties are automatically updated to display the current values.  Sometimes a field will be blank if you are at end of file or there are not enough bytes before the end of file for a valid value.

 

Changing Values

 

Most of the fields can be manually edited which changes the bytes following the cursor in the current file.  For example, to change a 32-bit integer value type the new value into the 32-bit field (after selecting the appropriate endianness and format), then press the Enter key.  If you do not want to make the change press the Escape key instead of Enter.  Of course, any changes made this way can be undone as normal, if you make a mistake.

 

Note that even in insert mode any changes overwrite existing bytes and do not increase the file length.

 

Sometimes fields may be disabled, ie changes are disallowed.  This can be due to several reasons such as the data being read-only, or there not being enough bytes before end of file.

File Properties Page to view information about the disk file

Character Values Page to view information about the byte/character at the cursor

Floating-Point Page to view floating-point values of the bytes following the cursor

Date/Time Values Page to view or change date and time in the bytes following the cursor

Status bar to view byte values and invoke the Properties dialog
