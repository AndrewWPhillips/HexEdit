Aerial View

 

Aerial View is a new feature in HexEdit 3.5.  It allows you to get an overview of even very large files at a glance then drill down to specific parts of a file.  There are also facilities for displaying the selection, search occurrences, etc (as marching ants or in the border) which is very useful for dealing with large amounts of information at a glance - for example you can use it to check when making a large selection that you are getting the part of the file that you expected.

 

Aerial View is also useful to reverse-engineer file formats for which you don't have access to the documentation (or the format was never documented).  For large files one of the first steps is to get an overview of the file by looking for areas that are different from others that may indicate special data like record headers.

 

The Display

 

When Aerial View is turned on (or a file is opened and Aerial View mode is on by default) HexEdit prepares a bitmap (ie, graphical image) of the file where each byte (or group of bytes for very large files) is represented by one pixel.  This happens in the background (ie in a low-priority background thread) so will not affect anything else you want to do with the file.  The Aerial View of the file is displayed in a separate "view" pane (similar to the template view) which can be shown in a split pane to the right of the hex view or in a separate tabbed window.

 

Note that for a large file there might be a delay while HexEdit scans the file but often the Aerial View appears almost immediately.  While the scan is taking place you can continue working but the aerial view will display as gray the part of the file not yet scanned.

 

It is very important to understand how the bitmap appears in the Aerial View pane is affected by these two things:

 

1. The number of bytes per bitmap pixel. Generally there is one byte per pixel but for very large files (approaching or exceeding the GByte size) then HexEdit needs to create a bitmap where each pixel represent 2, 4, 8 or more bytes.  Note that the colour of this combined pixel is the result of the combined colors of each of the bytes so there is still a lot of information being displayed.

 

2. How big a bitmap pixel is displayed on the screen.  On some monitors nowadays a single pixel is almost imperceptible so HexEdit allows you to zoom (see below) the bitmap so that a single "pixel" is displayed on screen in a block of pixels as large as 16 by 16.

 

Zoom

 

There are several abilities of the aerial view window that make it very useful.  First you can zoom in and out on the view to see more detail, using Ctrl+mouse wheel as usual or using the context menu. If the bitmap is too large to display then you can also scroll up and down using the scrollbar or the mouse wheel.  Also note that the bitmap width adjusts to the width of the window so you can often see new patterns by changing the window width which changes how different pixels align in the bitmap.

 

Colors

 

To make best use of the aerial view it is important to use a suitable color scheme.  If you are unaware of it, HexEdit (since version 2.1) has had the ability to set up color schemes where ranges of bytes can be displayed in different colors.  For example if the a certain byte value or ranges of values are important then you can set those to display in red. See the Color Options page for details.

 

If you know nothing at all about the data then using the "rainbow" color scheme that comes with HexEdit is a good way to see the spread of byte values.  This splits the 256 byte values into 12 ranges of about 20 bytes each with the colors for each range varying from red, through various shades of green, blue, etc to purple.

 

With practice you will learn to recognize what different patterns indicate.  Also changing the width of the aerial view window can reveal new patterns by allowing different pixels to come into alignment

 

Other Display Features

 

The Aerial View is also very useful for working with a file in other ways since it shows information about the selection, bookmarks, highlighted bytes, search occurrences.  You can see them in the aerial view as "marching ants" and or in the left margin.

 

Marching ants are dotted lines that slowly move like a trail of ants. They enclose the relevant bytes and slowly move to make it easier to see the boundaries.

 

For example, imagine you have performed a search and the search occurrences pane of the status bar informs you that there are thousands of occurrences of the search text.  If you want to get an overview of where throughout the file these occurrences are you can simply inspect the aerial view and looking for searches occurrences shown as marching ants and/or in the left margin in the current search color (green by default)).

 

Marching ants and the left margin indicators can be shown individually for each of:

 

1. selection - shown in black and white

2. the mark - cyan in default color scheme

3. bookmarks - blue in default color scheme

4. highlighted bytes - normally yellow

5. background search occurrences - green

 

To turn on/off these options use the context menu by right-clicking anywhere in the aerial view window.
