Calculator

 

 

HexEdit includes a built-in calculator similar to the Windows calculator but with some enhancements and integration with the rest of HexEdit that make it very powerful.  The calculator can be used in two ways:

 

1. As an integer calculator supporting integers of any precision and in any radix (base) up to 36.  Most of the calculator buttons perform integer operations.

2. As an expression evaluator supporting C-like expressions with brackets and precedence, with 5 data types (integer, real, string, boolean and date), functions, variables etc.

 

How to perform integer operations and evaluate expressions is explained further below.

Display

Edit Box

The heart of the calculator is the main edit box which allows you to enter numbers (and expressions) and view results.

 

Unlike the Windows Calculator you have full control of the text in the edit box.  For example you can edit a result, copy and paste all or part of the values, etc

 

When a date value is displayed it is converted to a string and wrapped in the DATE() function (which converts a string back to a date).  This is necessary as there are no date "literals" supported by expressions.

 

History

The down arrow at the right end of the edit box displays a history of previous calculations.  You can select an entry to redo the calculation or just view previous calculations.

 

Hold the mouse over any history entry to see the result of the calculation.

 

Operator/Status Indicator

To the right of the edit box is an area that may display:

  • the binary operator in effect if one of the binary operation buttons has been selected (+, -, *, etc)

  • O if the last integer operation generated an overflow

  • E if the last operation caused an error, such as divide by zero

  • R if the last operation was an expression which generated a real (floating point) result

  • S if the last operation was an expression which generated a string result

  • B if the last operation was an expression which generated a Boolean result

  • D if the last operation was an expression which generated a date

Current Expression

Above the edit box is a small area that displays the current expression (as it would be evaluated at that moment).  This is very useful when using the unary and binary operation buttons if you can't remember what buttons you have already used.  It also makes it plain that the buttons do not support any operator precedence.

 

If you wish to evaluate an integer expression with the normal mathematical precedence type the full integer expression into the calculator.  Expressions also support brackets.

 

Menu Buttons

At the top of the display are buttons that allow you to recall various values.  The Hex Jump History and the Decimal Decimal Jump History lists allow you to select from the histories of the Jump Tools (see Toolbar Tools).

 

The Variables menu allows you to recall previously assigned variables or clear all variables.  Variables can be assigned in the calculator (or in the Jump Tools) by using an expression involving an assignment (ie, using the '=' operator as in any C/C++ expression).  Variables may have any of the five types supported by expressions.

 

Depending on the radix in use there may be some ambiguity between an integer value (such as FFFF in hex) and a variable name.  To avoid any ambiguity any variable name may be preceded by the '@' character.  For example, @A is always taken as a variable with the name A, and never as the number 10 in hex.

 

The Functions menu allows you to select from the predefined functions that are supported by expressions. Functions are grouped according to the type of the values they return.  Note that, the predefined real constant PI is also defined here.

 

To make full use of functions you need to read the documentation.  For example, the min() and max() functions can take any numbers of parameters (except zero), whereas the menu implies they may must take two.

 

Bit Display

Below the Edit Box is the Bit Display which visually shows the state of the bottom 64 bits of the current calculator value (if it is an integer). Click any of the bits to toggle it on or off.

 

Radix, Bits, Signed

The Radix, Bits and Signed controls allow you to change how integers are entered and displayed.  They are discussed in detail below.

 

Buttons

Most of the calculator is taken up by the buttons as typically seen in calculators.  The layout of the buttons is reminiscent of the Windows calculator but a little better organised:

  • digits buttons in the middle (gray)

  • binary operations on the right (blue)

  • unary operations on the left (purple)

  • file operations on the far left (black)

  • memory and mark operations at top-left (red)

  • control buttons (Clear, Equals, etc) at top-right and bottom-right (red)

Most of the buttons are only used with integer operations not with expressions.  Their behaviour is described more fully below.

Operations

Expressions

 

Expression evaluation is very convenient for entering an expression then pressing enter (or the '=' button) to get the result.  Expressions are used throughout HexEdit such as in the jump tools and templates.  Using the calculator makes it easy to test expressions, especially as it has a convenient drop-down History list that allows you to recall and modify previous expressions.

 

Unlike when using the calculator buttons expressions supports precedence, saving values to variables (using '=' operator), as well as floating point numbers, date calculations, functions etc.  There are drop-down menus allowing you to select any of the available variables or predefined functions.  Variables also support arrays.  An example of an expression you could use is:

 

R[N] = rand()/0x10000 + min(P, Q, R)

 

Note that x is not used for multiplication -- use an asterisk (*) instead.  The characters "0x" as seen above is used to precede a number to ensure it is treated as a hex literal.

 

Most of the calculator buttons are designed to work with integers, and you can use integer expressions and combine these with calculations using the buttons.  However, if you evaluate an expression that gives a result of another type the result is displayed but cannot be combined with the integer (button) operations. To remind you of this a character is displayed next to the result of non-integer expressions:

 

R = real or floating point value

S = string

B = boolean

D = date

 

Integer Operations

 

Most of the buttons are designed to support integer operations, because the calculator in HexEdit is primarily used with integers such as file addresses.  The rest of the discussion focuses on these integer operations.

 

Note that you can combine the use of integer expressions with using the buttons.  In fact it is often more convenient to use an expression than to use the buttons as expressions support precedence and all operations of the buttons can be performed with an expression - for example, the & operator duplicates the "AND" button (see Expressions or a C reference manual for information on all operations).

Buttons

Binary Operators

 

To perform a binary operation, enter the first operand (value) and click the desired operator button. You can still click a different operator if you accidentally clicked the wrong one. A small indicator appears to the right of the numeric entry box that indicates the current operator in use. After you enter the second operand you can then click the result button (=) or another binary operator to generate the result. If the operation generates an overflow or error then the calculator beeps and an E or O is displayed next to the numeric entry box.

 

As well as the normal arithmetic operators, and logical operators there are a few more esoteric operators. The LSL (logical shift left) and LSR (logical shift right) operators simply shift all the bits in the first operand (left or right) by the number in the second. The new bits added to the end always have a zero value.

 

The ASR (arithmetic shift right) operator, on the other hand, will add new bits with the value one if the current top bit is one. Whereas LSR is equivalent to a division by a power of 2 for an unsigned number, ASR is equivalent to a division by a power of 2 for a signed (2’s complement) number.

 

ROL (rotate left) and ROR (rotate right) are similar to LSL and LSR but the new bit shifted in is the bit shifted off the other end.

 

The GTR and LESS buttons find the larger or smaller of their two operands. These are useful in keystroke macros, particularly as the macro error level (see below) is set to a warning if the first operand is not greater (for GTR) or not less (for LESS) than the second.

 

The division operator (/) gives the truncated integer division, whereas the MOD operator gives the remainder. These are equivalent to the C/C++ integer / and % operators.

 

Unary Operators

 

Unary operators take a single operand -- the currently displayed number. Any current binary operator is not affected by a unary operation. All unary operator buttons are to the left of the numeric buttons except the change sign operator (+/-) that is between the 0 and = buttons.

 

There are arithmetic operators for square, factorial etc. Note that the square root operator truncates the result to the integer less than the real result. These operators can generate error and overflow conditions, which causes an E or O to be displayed next to the numeric entry box.

 

The ROL1, ROR1, LSL1, LSR1, and ASR1 buttons perform the same operation as their binary counterparts but shift by one bit only.

 

The INC and DEC buttons increment and decrement their operand. If this results in an overflow for the current operand size (see the discussion of Bits below) then a macro error is generated.

 

The REV button just reverses the order of all the bits in the current number. So if you are operating on 32 bit values bit 31 is swapped with bit 0, bit 30 with bit 1 etc. The FLIP button flips all the bytes in the current number. If you are working with 8 bit values this button is disabled since there is only one byte.

 

The @n operator takes the current number and uses it as an address into the currently active file. Using that address it reads the (8, 16, 32 or 64 bit) value from the file at that address. It is a unary operator as it takes an address and replaces it with the value at that address. See the File Access section below for more information.

 

Radix, Bits and Signed

 

The Radix buttons allow you to enter numbers and display results in Hex, Decimal, Octal or Binary, and convert between those bases. Note that when you select Decimal mode the numbers default to being signed.  The Radix edit control allows you to select any radix (base) from 2 to 36.

 

When using numbers in a radix greater than 16 the digits are G (16), H (17), ... Z(35).  There are no buttons for entering these digits so you must use the keyboard.

 

The Bits buttons allow you to specify the size of the numbers you want to work with. Note that if you change to a smaller number of bits and the current value won’t fit this will generate an overflow. For example, if the current value is 257 and you click on the 8 bit button an overflow is indicated and the current value changes to 1 (the value in the low byte of 257). To restore the original value increase the number of bits back (eg. click on the 32 bit button).  You can set any positive value for bits; using a value of zero (or selecting the Inf radio button) gives infinite precision integer calculations..

 

Note that generally integer operations are performed using infinite precision, and then displayed with the requested number of bits.  If the number is too large to be displayed in the number of bits then an overflow is indicated.  You may then increase the Bits setting to see the full result.   For example, if you add 255 and 2 in 8 bit mode an overflow is generated and the displayed result is 1. If you now increase the Bits setting you can see the actual value of 257.

 

The Signed setting allows viewing a number as unsigned or signed (2's complement) numbers in any radix - but Decimal mode defaults to signed, while other modes default to unsigned.  Switching between signed and unsigned can generate an overflow if the high bit is on.

 

  The Signed checkbox is disabled in Inf precision mode as signed numbers are always used in this case.

 

File Access

 

The buttons to the far left (except for the memory buttons) are only enabled when there is a file window active. They allow you to get the value of and even change the cursor, mark etc in the currently active file. Further, you can read and write values in the file.

 

The farthest column of buttons (Memory, Mark, @Mark, etc) are used to get values, whereas the buttons immediately to their right are used to change the corresponding value using the currently displayed number. There is also a unary operator button (@n) that can be used to get a value from the file – it is discussed above under Unary Operators.

 

The GO button (in the bottom right corner) is similar to the Set Cursor button but is also puts you back into file editing. This is the “default” button, so that if you type a number and press the Enter key, you will jump straight to the address you typed and can continue editing.

Note that there is no “Set EOF” button as you can’t change the length of the file in the calculator. (You can only change the length of the file if you are in INS (insert) mode and you add or delete bytes at the end of the file.)

 

The size of the integer read by the “@” buttons (@Mark, @Cursor, and @n) and written by the Store buttons is determined by the current Bits setting. The byte order of the numbers read from or written to the file is set by the Big-endian check-box near the top right corner of the calculator. (This is normally not checked allowing for Intel byte order.)

 

If you are not editing any files, the file access buttons are disabled (light gray), because there is no file to access. Some of the buttons may be disabled at various times for various reasons. For example, if the current displayed value is greater than the length of file then MkS, @n etc, that use that value as a file address, are disabled.

 

The file access buttons are particularly useful when used with keystroke macros (see previous section). For example, you could repeat the same calculation on every value in a file by storing the following operations in a keystroke macro: first load the value at the cursor using the @Cursor button, perform some operations on it, then save it back to the file using the Store button. To move to the next value in the file: load the current cursor address using the Cursor button, increment the required number of times depending on your operand size (using the INC button) and move the cursor using the Set button. Use the Tools/Macro Play menu item to replay the macro the required number of times (use a very big number to do the whole file).

 

Memory

 

To store the displayed number in the calculator memory, click the MS button. After a number has been stored, the Memory button is enabled, which you can later click to get the number back. The MC button clears the memory (and disables the Memory button). Note that you can generate an overflow just by recalling a number from the memory that is bigger than will fit in the current operand size.

 

The M+ and M- buttons allow you to add to and subtract from the current value in the memory.

 

The buttons below the memory buttons for the “mark” are analogous, but instead of changing the calculator memory they change the position of the mark in the active file window. They can be used as a makeshift second memory.

Shortcut Keys

The calculator provides many shortcut keys that are the same as those for the Windows calculator. For example, to recall the value of the calculator memory: press Ctrl+R. The shortcut keys are displayed in the “tips” that appear when you leave the mouse over a button for a second. The most common ones are for arithmetic operators: +, -, * (multiply), / (divide), and % (modulus); logical operators: &, | (OR), ^ (Exclusive-OR), and ~ (NOT); and = (Calculate).

 

Like the Windows calculator you can enter numbers by typing valid digits for the current base. The number displayed has spaces added for readability, but you don’t need to type these. Decimal numbers have commas as thousands separators (or whatever is set in the Control Panel for your culture/region).

 

Unlike the Windows calculator you can use the Home, End, Del, arrow keys etc to edit the current number. You can even select parts of the value using the Shift-arrow keys before deleting, cutting (Ctrl+X), copying (Ctrl+C) etc, and use Ctrl+V to paste from the clipboard.

 

Particularly useful are the keys to change the current base and size: F5 to F8 change betweens the four bases; F9-F12 change between the four sizes (64, 32, 16 or 8 bit).
