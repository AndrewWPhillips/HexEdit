Bookmarks

 

A bookmark associates a name with a file and a location within that file.  Most operations with bookmarks are performed using the Bookmarks dialog:

 

 

For example, type a name and click the "Add" button to add a bookmark in the currently active file at the current cursor position.  You can also perform other operations on bookmarks, such as deleting them, by selecting one or more in the list and clicking the "Delete" button.

 

To jump to a bookmark just select it in the list and click the "Go To" button, or just double-click it.  If the file containing the bookmark is not open then it is opened and the cursor is positioned at the bookmark location.  You are warned if the file is not found on disk or the bookmark is past the end of file.

 

You can validate all the bookmarks in the list by using the "Validate All" button.  If you have bookmarks you want to retain that are in files that may not always be available (ie on a network drive or removeable media), you should select the "Keep network & removeable" check box.

 

Each bookmark has several attributes stored with it, which are displayed in the columns of the bookmarks dialog.  To toggle any column on or off right-click on a column heading and select from the context menu.

 

The ability to sort the columns  is useful for finding a bookmark for which you have forgotten the name.  For example, if you can remember when you created it you can sort by the "Modified" column, or when you last jumped to it sort by the "Accessed" column.

 

Other Uses for Bookmarks

 

Bookmarks are also integrated with other parts of HexEdit.  Here are some other ways they are used.

 

In the Find Dialog you can bookmark all search text occurrences using the "Bookmark All" button.  You specify a bookmark prefix and HexEdit appends a number to this prefix to create a unique bookmark name for all found occurrences.  This is useful if you want to remember all occurrences of some search text before making changes that might change some of them.  Remember that you can also search several files using a search scope of "All open files".

 

Bookmarks for the current file can also be used in expressions in the hex and decimal jump tools.  For example, you can type an expression such as "base+5" into a jump tool to jump to an address 5 bytes past the bookmark called "base" (assuming that bookmark exists in the current file).

 

If you want to use bookmarks in the hex jump tool remember to not use names for bookmarks that could also be a hex number (like "aaa").  Also remember that bookmark names are case-sensitive.

 

Other Bookmark commands

 

Often it is tedious to have to open the Bookmarks dialog to delete or add a bookmark.  A quick way to do this is to use the Toggle Bookmark command.  If the cursor is positioned at a bookmark then the bookmark is deleted, otherwise a new bookmark is created (with a name like "unnamed005").  You can use this very much like the toggle bookmark command in Visual Studio (it is mapped to Ctrl+F2 as in Visual Studio), but it has the added advantage that the bookmarks are not lost when you exit the program.  If you do want to remove all "unnamed" bookmarks go to the Bookmarks dialog and sort by name the select and delete all bookmarks starting with "unnamed".

 

It is common to want to look at all bookmarks in a file by stepping through them.  To this end the Next and Previous Bookmark commands are very useful.  These are mapped to F2 and Shift+F2 as in Visual Studio and other programs.

 

To jump to a bookmark by name you can select from the drop-down list of the "Bookmark tool".  By default this tool appears on the Navigation Toolbar.  This tool is also updated as you move the cursor to show the closest bookmark above the current address.

 

To remove all bookmarks in the current file just use the Clear Bookmarks command.  If you just want the bookmarks to be hidden but not deleted use the "Hide Bookmarks" toggle.
