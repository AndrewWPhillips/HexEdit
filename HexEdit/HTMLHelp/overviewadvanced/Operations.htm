Operations

 

Most of the commands on the Operations menu are used to perform arithmetic or bit manipulation operations on the byte (or word, double-word or quad-word) at the cursor.

 

These commands also work on several bytes (or words, double-words or quad-words) if there is a selection.  This provides a quick and easy way of performing the same operation many times.  However, more complicated operations may only be accomplished by using a keystroke macro and the calculator.

 

Binary Operations

 

All of these operations take the value from the file manipulate it and write it back.  But some of them (eg, addition, subtraction, AND etc) require a second operand.  In this case the current value in the calculator is used.  This is done to facilitate the use of operations in keystroke macros.

 

Note that the size of the operand in the calculator must be the same or smaller than the operand required for the operation.  If you need to perform a double-word (32-bit) binary operation then the calculator cannot be in 64-bit mode.

 

Operand Size

 

Most of the operations provide byte, word, double-word and quad-word versions.  For some operations this is inappropriate -- for example, a bitwise NOT operation does not change depending on the operand size.

 

If operating on more than one value the current selection length must be a multiple of the operand size.  For example, for a double-word operation the selection length must be a multiple of 4 bytes.  You can easily check this by holding down the shift key, which not only tells you the selection length but the largest binary divisor of the length of the selection in square brackets.  For a double-word operation the number in the square brackets must be 4, 8, 16 etc.
