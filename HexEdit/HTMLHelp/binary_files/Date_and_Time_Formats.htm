Date and Time Formats

 

Dates and times when stored in a binary format are usually stored as a numeric value. Often the date and time are stored together in one value usually a 32 or 64 bit integer.  Typically the operating system determines the format of dates, but sometimes specific compilers use their own representation, and convert to the OS format when necessary.

 

For example, C/C++ compilers use a numeric value called a “time_t”. Although a time_t could be any sort of numeric value (depending on the compiler) it is typically a 32 bit signed number representing the number of seconds since midnight of 1st January 1970. (This is usually the format for UNIX and Windows compilers.)  The exact representation is usually irrelevant even to the programmer, but if the time_t value is written to a binary file it is important to your investigation of the file to know its format.

 

An important use for storing date/times is when the operating system keeps track of the time a file was last modified (or accessed, created etc).  MSDOS file date/times are stored as separate years, months, days, hours, minutes and seconds each as a number of bits (or bitfield) in 4 bytes. (Actually seconds is only stored in 5 bits giving a resolution of 2 seconds.)  In contrast the NTFS file date/times used by Windows XP/2000/NT are stored in a 64 bit integer in 100 nanosecond increments.

 

Another common format is to store days since a certain time in a floating point value. Thus the fractional part stores the time (eg. 0.5 means midday). For example, Microsoft OLE date/times are the number of days since 30th December 1899. This has the advantage that times close to the present can be stored accurately and distant times can still be represented (if less accurately).
