Boolean Operations

 

A group of bits can be used to store a number as we saw in the previous section.  Moreover these numbers can be manipulated by hardware to perform arithmetic operations as we will see later.  However, there are other basic operations that can be performed on bits and groups of bits which we will explore in this topic and the next topics.

 

Note that processors rarely provide operations on individual bits, but operate on groups of bits of byte size and higher.  In the following discussion we will talk about byte operations, but operations that work on 16-bit and larger values are common and behave analogously.

 

When performing boolean operations on bytes, words etc each bit is operated on independently.  In other words there is no interaction between the bits as there is in addition or shifts.

 

Boolean arithmetic has some similarities to arithmetic with numbers but there are some significant differences.  When performing numeric arithmetic the possible operands are infinite; when performing boolean arithmetic there are just two possible values: "TRUE" and "FALSE".  (Actually we are just dealing with a special case of a more general Boolean algebra often called "set theory" where the number of operands is infinite.)

 

The most important boolean operations are called AND, OR and NOT.  These have meanings somewhat like their meaning when used in the English language.  However, as English can be imprecise we need to define their meanings exactly.

 

The NOT Operation

 

The simplest boolean operation is NOT.  It takes one operand (hence it is called a unary operation) and produces the opposite value.  Hence NOT(TRUE) is FALSE and NOT(NOT(TRUE)) is TRUE etc.

The AND and OR Operations

 

The boolean operations AND and OR take two boolean values and produces a boolean result.  They are called binary operations, for the same reason that addition is called a binary operation.  That is, they are "binary" because they take two operands not because they operate on binary values.  To avoid this confusion they are often called "dyadic" operations which again simply means they take two operands.

 

The AND of two values is TRUE only if both values are TRUE; if either or both are FALSE then the result is FALSE.  Similarly the OR of two values is FALSE only if both values are FALSE; if either or both are TRUE then the result is TRUE.

As an example consider the statements:

 

    A. Daniel is big.

    B. Daniel is strong

 

Hence the statement "Daniel is big AND strong" (which in English is a shorthand for "Daniel is big AND Daniel is strong") is only true if Daniel is both of these.  But the statement "Daniel is big OR strong" is true if he is either (and possibly both) of these.

 

If you look closely at the table above for AND you will notice that AND is similar to multiplication if you consider TRUE to be one and FALSE to be zero.  Similarly OR is like addition (except that 1+1 is 2).  When evaluating numeric expressions traditionally multiplication has a higher precedence than addition, hence in a boolean expression AND is considered to have higher precedence than OR in many programming languages such as C (and hence in HexEdit Pro's expression evaluation).  So in the statement:

 

     A AND B OR C AND D

 

the AND operations are evaluated first:

 

     (A AND B) OR (C AND D)

 

De Morgan's Theorem

 

Boolean arithmetic is unlike numeric arithmetic in many ways.  For example, there is an interesting symmetry between AND and OR -- if you swap TRUE and FALSE in the above tables then all operations are the same if you also swap AND with OR.

 

De Morgan's Theorem is another with no analogy in numeric arithmetic.  It simply states that:

 

    NOT(A OR B) is equivalent to NOT(A) AND NOT(B)

 

And, by symmetry:

 

    NOT(A AND B) is equivalent to NOT(A) OR NOT(B)

 

You can verify this theorem yourself by considering all possibly values in the above boolean expressions.

 

An example, of De Morgan's theorem is the statement:

 

    Daniel is NOT big OR strong

 

which is equivalent, by De Morgan's Theorem, to:

 

    Daniel is NOT big AND Daniel is NOT strong.

 

De Morgan's theorem is very useful in simplifying boolean expressions.

 

The Exclusive-OR Operation

 

In English the meaning of OR can be ambiguous.  It can mean OR as discussed above but more often means exclusive-OR.  For example, consider the statements:

 

    A. Emma went by bus.

    B. Emma went by train.

 

The statement "Emma went by bus OR train" would typically mean that she took either the bus or the train but not both.  This is called exclusive-OR since one statement being TRUE automatically excludes the other.  In other words, Emma can't travel by bus and train at the same time, these conditions being mutually exclusive.

 

To differentiate the normal OR we discussed above from exclusive-OR it is sometimes predictably called inclusive-OR.  In English, if you want to use inclusive-or to explicitly include both conditions then this is often written as "and/or".  For example, "Emma went by train and/or bus", which means she could have taken both (presumably changing from one to the other at some point in the journey).

 

Exclusive-OR is often abbreviated to "XOR" or "EOR".  Similarly inclusive-OR may be abbreviated to IOR.  However, plain OR always means inclusive OR.

An interesting property of the exclusive-OR operation is that there is no lost information, unlike when AND or OR is used.  For example, if you XOR a byte with a certain value you can get back the original value by simply XOR'ing again with the same value.  In other words:

 

   (A XOR B) XOR B  is equivalent to A

 

This property has uses in many areas such as encryption and drawing.  For example, it is common to draw cursors in XOR mode so that to erase them it is simply a matter to draw them again.

You can also use XOR for flipping selective bits in a byte.  This is done by simply XORing with a byte that has the bit on for all bits to be flipped and the bit off for all bits to be unchanged.
