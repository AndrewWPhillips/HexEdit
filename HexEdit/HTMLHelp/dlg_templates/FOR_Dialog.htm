FOR Dialog

 

 

This dialog allows you to set the options of a template FOR element (and its contained element, using the Edit button).  It is usually invoked by right-clicking the FOR node in the template tree view and selecting the "Edit..." item.

 

The FOR element is used to create a repetition (ie, zero or more occurrences) of its sub-element.

 

Name

This name identifies this specific field in the file.  The name is used in expressions to access array elements (using an index in square brackets).  The name is required (unless the parent element is also a FOR) so that you can access this element, and should be different to the names of siblings (if the parent is a STRUCT).

 

Count

This is the number of elements of the array.  The value may be a constant integer or an expression that evaluates to an integer.  You can use any value from the template so far processed in the expression.  Use the button with the triangle to see a menu (and possibly submenus) of all available values.

 

If the expression evaluates to zero for a particular data file then the array contains no elements.  However, even in this case a sub-element is shown (greyed out) in the template tree view - this allows you to edit the sub-element of the FOR, even if the array is empty.

 

Stop

A boolean expression that returns true when the array should terminate.  The expression will use the special variable "this" to get the current array value.  Use the button with the triangle to see a menu (and possibly submenus) of all available values including "this".

 

As an example: if the element contained in the FOR is a DATA element of integer type then an expression like "this == 0" could be used, whence the last element would be zero.  Another example: if the contained element is a STRUCT which itself contains a DATA element with name "flag" then an expression like "this.flag == -1" could be used.

 

Note: If both Count and Stop then the array stops when one or the other conditions are satisifed. If neither are specified then the array continues to end of file (in which case there should be no more elements after the array).

 

Element

This displays the name and/or type of sub-element contained in the FOR.

 

Edit

Use this button to edit the element that is contained in the FOR.  You cannot insert or delete elements like in a STRUCT since the FOR must always have exactly one sub-element.  However, you can replace it with an element of a different type (see below).

 

Replace with

Use this button to replace the element contained in the FOR with an element of a different type.

 

Type name

Generic type name for the data element.  For example, this could be the name of a C/C++ typedef.  This is a comment field only and has no effect on the behaviour of the template.

 

Comments

Add any other relevant information here.

 

<< and >> buttons

If the parent element is a STRUCT then these buttons are displayed in the bottom right of the dialog.  They allow you to move to the previous and next sibling (ie the previous and next elements in the parent STRUCT).  This is very handy when you quickly want to view or change values in all the children.

 

Using the button is equivalent to clicking OK then editing the previous or next element in the parent STRUCT.
