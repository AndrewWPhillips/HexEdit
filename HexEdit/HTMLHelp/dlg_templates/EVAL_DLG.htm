EVAL Dialog

 

 

This dialog allows you to set the options of a template EVAL element.  It is invoked by right-clicking the EVAL node in the template tree view and selecting the "Edit..." item, or by clicking an Edit, Insert, or Replace With button in the dialog for the containing STRUCT, or Structure definition.

 

An EVAL is useful for evaluating expressions (eg initialising variables), for displaying relevant information to the user of the template, or for indicating some sort of error condition.

 

Expression

An expression using any template field so far encountered, variables etc.  Expressions can be of boolean, floating point, boolean, date or string type.  To display a simple message just use a string constant - ie characters enclosed in double-quotes with possible escape characters (as in C/C++).

 

A special feature when using a string is to be able to include any number of expressions, with optional format specification.  An expression is simply enclosed in braces {} with an optional semicolon (;) followed by a format specification appropriate to the type of the expression.  For example, to display fields from a STRUCT you could use an string like this:

 

    "X = {locn.x; %g}, Y = {locn.y; %g}, flags = {locn.flags; bin}"

 

See the section called Displaying Information in the Tree Column in Editing templates for more information.

 

Display error if expression evaluates false

If checked the above expression must return a boolean result.  (If the value is not boolean an error is displayed and the user can terminate template evaluation.)  If the return value is false the node is shown in the tree view and expanded so the user will see it.

 

Always display result in tree view

If checked the result of the expression is always displayed as a node in the tree view, otherwise it is only displayed in edit mode.

<< and >> buttons

If the parent element is a STRUCT then these buttons are displayed in the bottom right of the dialog.  They allow you to move to the previous and next sibling (ie the previous and next elements in the parent STRUCT).  This is very handy when you quickly want to view or change values in all the children.

 

Using the button is equivalent to clicking OK then editing the previous or next element in the parent STRUCT.
