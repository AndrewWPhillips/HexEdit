Selecting and the Clipboard

 

To select part of a file you can use the mouse or the keyboard. To use the mouse, click the first byte you want to select and drag the mouse to the last byte. Alternatively, you can click (and release) on the first byte then shift-click (hold down the Shift key while you click) the last byte. Shift-clicking is very useful if you need to select a large area since you can use the scroll bar to quickly move to the other end of the desired range.

 

To use the keyboard you can use the Shift key with the normal cursor movement keys. For example, to select everything from the current cursor position to the end of the file press Shift+Ctrl+End.

 

Once selected you can cut or copy the bytes to the clipboard in the normal manner. Note that you cannot cut if the window is in either of the read-only or overtype modes since this will delete bytes from the file. You can also perform many other operations with selected text such as write it to a file, print, compress, export etc.

 

Selection Length

 

While you are making a selection a small "tip" window is displayed which displays the length of the selection (in hex or decimal depending on how addresses are currently displayed).

 

You can bring this tip window up at any time by holding down the "Shift" key.

 

There is also an option for showing what power of 2 the selection length is divisible by.  This is useful as some commands only work with a selection length which is a multiple of 2, 4 or 8.  This option can be turned off in Workspace Display page of the Options dialog.

 

Highlighter

 

The highlighter has already been mentioned but it can also be viewed as allowing a disconnected or discontiguous selection (ie all the selected bytes do not have to be next to each other).

 

You can use the highlighter to mark particular important bytes within a file.  Clicking the highlighter, highlights the selected bytes.  To quickly create several highlights, turn on highlighter mode (click the highlighter button when there are no bytes selected), thence all subsequent selections made will be highlighted.  Highlighter mode can be used when comparing files to highlight all differences that were found.

 

Clipboard Formats

 

You have a choice of how HexEdit Pro works with the clipboard, but first you need to understand how data is stored on the Windows clipboard.  There are actually three different ways that are used depending on the circumstances:

  1. Data is copied to the clipboard in two different formats at the same time.  This is how the clipboard has been used since HexEdit 1.0.

    • binary format (the same as used in Visual Studio hex editor) - this allows copying and pasting with no loss of data (unlike text formats which cannot handle all byte values)

    • text format, converted from the current character set as necessary - this allows copying and pasting with text editors, word processors, etc

  2. Data is copied to the clipboard as hex text.

    • text format, where each byte is stored as two hex digits (plus space and end of line characters) - data can be copied and pasted in HexEdit Pro (with no lost information) and pasted as hex text into a text editor

  3. If the data is too large for the clipboard to handle a special clipboard format is used where only a small amount of information is held in memory and the actual data is kept in a temporary file.

    • custom format, where the actual data is stored in a temporary file - allows copying and pasting of selections of any size (subject to disk space), but the data cannot be pasted into other software

For large data the last format is always used, but whether 1 or 2 is used depends on settings chosen in Workspace Editing Options.

 

There are also other ways to interact with the clipboard, all of which work with text.  First, you need to understand that text on the clipboard is stored internally as Unicode (in versions of Windows derived from NT).  When you copy or paste text data in HexEdit Pro the text will be automatically converted according to the current character set.  If you are using EBCDIC then anything you cut or copy to the clipboard will be converted from EBCDIC to Unicode.

 

When pasting from the clipboard the reverse conversion is supplied, so if you are pasting from a Windows text editor into HexEdit Pro in EBCDIC mode the text would automatically be converted from ASCII or Unicode to EBCDIC.  However if you are copying and pasting within HexEdit Pro then the binary format data is used in preference to the text data and hence no conversion is performed.  If you want to copy and paste within HexEdit Pro and convert between character sets you need instead to use one of the Paste As ASCII, Paste As EBCDIC, or Paste As Unicode Text commands.

 

The Copy As C Source allows even more sophisticated ways to copy text to the clipboard. This was designed for initialising many different sorts of array in C, C++ and other programming languages, but it is far more useful than that - for example you can create CSV integer or floating point data suitable for importing into other programs such as a spreadsheet.

 

Working with Large Files

 

The clipboard is great for working with small to medium sized blocks of data, but will not work for blocks of more than about 16 Mbytes.  Furthermore, when you paste data into HexEdit Pro the pasted data is stored in memory.  If you paste a lot of large blocks of data you will cause Windows to slow down or even crash through memory exhaustion.

 

The Append to File and the Append to Same File commands were created to avoid these problems.  With these commands you can easily build multi-Mbyte or even multi-Gbyte files easily, by building a file from large blocks appended to a file in order.

 

Another problem with large files is that the scrollbars are virtually useless.  In a multi-Gbyte file there are many Mbytes between each "notch" of the scroll bar, and it can take hours to page to the address you want.  Of course, you can easily jump to an address with one of the Hex or Decimal Address tools or the Go To command.  But how do you select a large block since you need to use the vertical scroll bar before shift-clicking?  The solution is to jump to one end (using a jump tool or the Go To command) and mark the position (using the Mark command), then jump to the other end and use the "Extend to Mark" command.
