Modeless Dialogs

 

 

Modeless dialogs are dialogs that are always available.  You do not have to close them to continue using other parts of HexEdit.  There are currently four modeless dialogs (calculator, bookmarks, properties and find dialog).  The modeless dialogs have been completely reimplemented in version 3.0 to be dockable, rollable etc -- they now behave somewhat like toolbars and may be referred to as dialog bars.

 

The modeless dialogs are very convenient for many uses.  Here are some examples:

  • check the value of bytes, numbers, dates etc at the current cursor position as you move it in the file using the Properties dialog

  • change the value of bytes, integer numbers, floating point numbers, dates etc in the Properties dialog

  • calculate an address in the Calculator and click the Go To button to quickly jump to it

  • quickly jump to a bookmarked position in any file (which does not have to be currently open) by double-clicking it in the Bookmarks dialog

  • quickly add a bookmark by typing its name and clicking the Add button in the Bookmarks dialog

  • type some byte value to search for and click the Find button in the Find dialog

By default, modeless dialogs "float" in front of the HexEdit window.  You can drag them by their title bar but you cannot force them behind the main HexEdit window.  This can be annoying sometimes so there are several ways that you can move them out of the way:

  • hide the dialog by clicking its close button (the X in the top right corner) - the dialog is just hidden and any information is still there when you reopen the window

  • drag the floating dialog to another part of the screen - if you have multiple monitors you can drag them all to another monitor and HexEdit will remember their positions when reopened

  • resize the Bookmarks dialog so it takes up less screen space

  • drag the dialog to the left or right edge of the main HexEdit window to "dock" it there

  • "roll up" the dialog by clicking the little arrow button next to the X button in the top right corner

  • failing all that if you try to move the cursor under a floating dialog HexEdit will move it out of the way so you can see what you are doing

These facilities are described below.

 

Displaying and Hiding

 

You typically use a menu item to open the dialogs.  For example Edit/Find... opens the Find dialog at the Basic Find page, whereas Edit/Replace.. opens it at the Replace page, while Tools/Find Hex... opens it at the Hex Find page.

 

You can also toggle a dialog (show or hide it) using the Dialog Bars sub-menu at the top of the Toolbars menu.  This is visible at the top of the View main menu or when you right click on a toolbar or dialog bar.  For example, if the Calculator is not visible you can display it using the View/Tool bars/Dialog bars/Calculator menu item.  The View/Tool bars/Dialog bars also has checkboxes which show you which dialogs are (or should be) visible.  If a dialog is checked but you cannot find it you can use the Help/Repair/Restore Dialogs... command (see below).

 

Docking

 

The modeless dialog bars are similar to toolbars in that they can float or be docked.  However, by default they are not dockable as some existing users find the docking behavior confusing or annoying.  To allow them to dock use the View/Tool bars/Dialog bars/Allow docking menu item.  Note that dialogs can only be docked on the left or right side of the main window.

 

To dock a dialog just drag it to the left or right edge where you want it.  The dialogs may be docked on top of or next to each other.  Sometimes you don't want a dialog to dock when you drag it near a docking position - in this case hold down the Ctrl key while dragging and it will remain floating.

 

Rolling

 

If you are short of screen space you may not want to waste it by docking dialogs, but you don't want to close them.  An alternative is to roll them up so that just the title bar is visible.  To roll and unroll just click the little arrow button next to the close button.

 

You can't dock a dialog bar when it is rolled, but you can reposition it by dragging the title bar as normal.  I normally leave the calculator rolled up near the top of the window.

 

Help

 

All controls in the modless dialogs have context help, which appears as a small "tip" window.  Help for a control is available in several ways:

  • click the ? button in the top right corner of the dialog (the mouse cursor changes to include a question mark) then click on the control

  • select the control and press F1

  • right click the control and select What's This menu item from the context menu

  • for controls that have there own context menu (eg, a text box) the What's This menu item is not available - just right click the text description to the left of the control

To get more detailed help and an overview of the dialog, click the Help button..

 

Repairing

 

Sometimes you may not be able to find or use a dialog.  This could be because it is almost entirely off the edge of the screen, or is rolled up and not noticeable.  Problems can also occur in the docking system where dialogs are docked while rolled or docked at the top or bottom of the main window.  If this happens use the Help/Repair/Restore Dialogs... menu item which:

  • undocks all dialogs;

  • unrolls all dialogs;

  • restores the default sizes of resizeable dialogs; and

  • moves them to the top left corner of the main window.
