Step 4. Testing and Fixing

 

         

 

In this step of the tutorial we will experiment with some changes to the template and introduce bugs in order to see how they can be found and fixed.

 

1. Reopen the text file you closed at the end of the previous step.  HexEdit will automatically use the same template that you had open last time.

 

You can quickly open a file you recently had open by selecting it from the recent file list on the File menu.

 

Tracking Down Template Bugs

 

Now we will change our template specification slightly.  We add the assumption that the text file lines should be no more than 128 characters in length.  We can check that this is the case by adding an expression to the "domain" attribute of the DATA element.

 

2. Right-click the root node and select the "Expand All" menu item to quickly see all nodes of the template.  Now double-click any of the DATA nodes under the "lines" node to edit the DATA element.

 

Editing any DATA element under the FOR node will affect all elements under that node.  In other words, they are all instances of the same DATA element.

 

First we will make a deliberate mistake in our domain expression.

 

3. Change the domain field to be "STLEN(this) < 128", and click OK.

 



Figure 4.1 DATA Element dialog with error in domain expression.

 

 

You will get an error message something like "Unknown variable "STLEN".  Do you want to continue?".

 



Figure 4.2 Error message.

 

This error is just telling you that the template does not know what "STLEN" is, since we should have typed the name of the built-in string length function "STRLEN".

 

A new variable is created whenever you assign a value to a new name.  In other words, you do not need to declare variables.  However, if you try to use a variable before it has been assigned to you will get the above "unknown variable" message.

 

If you click No in the dialog (to not continue) the last node shown in the tree will be the one in error.  This is useful for quickly finding the trouble spot.  

 

DATA Field Validation

 

4. Double-click the DATA element again and correct the domain field to be "STRLEN(this) < 128", and click OK.

 



Figure 4.3 DATA Element dialog with correct domain expression.

 

The domain expression just checks whether the current element is valid, and is evaluated for every "line" of the file.  The special variable name "this" refers to the current data element.  Technically the domain expression could be any valid boolean expression (eg, "false"), but to be useful should use "this" in some way or other.

 

The special built-in function "STRLEN" will return the length of a string.  This can be a string variable or a string data field from the file.  It is the number of characters up to (but not including) the terminating character.

 

If the text file you are editing contains any lines longer than 128 characters then they will be highlighted with a different (gray) icon to indicate an error.  Whenever errors occur in a template they are shown this way and the node expanded so that they can be easily seen.

 

You can cycle through all errors in a template by holding down the Ctrl key and pressing the Left or Right arrow key.

 

A domain error would typically indicate that there is an error in the data of the file (unless there is a bug in your template, of course).  To find out what bytes of the file are associated with the DATA value you can use the "Sync" button which is the left-most button on the Navigation toolbar.

 

If the Navigation toolbar is not visible right-click any toolbar or the menu-bar and select it in the list.

 

Clicking on the line in error in the tree window then clicking the "Sync" button will highlight the corresponding bytes in the normal (hex) window.

 

If you get sick of clicking the "Sync" button just turn on "Auto-Sync" mode using the next button on the Navigation toolbar.

 



Figure 4.4 Navigation bar showing Sync buttons.

 

Tree Window Columns

 

You may also be able to work out where the problem is in the data file or template by turning on the display of extra columns in the tree window.  To display a hidden column right-click on the tree window header and select the column you want from the list.  Visible columns are already ticked.

 

5. Right-click the tree window header and select the "Data" column from the list.

 



Figure 4.5 Column selection list with Data column selected.

 

You can also use the list to hide columns again.  But you cannot hide the tree (left-most) column - it does not appear in the list.

 

The "Data" column will show the data from the file associated with an instance of a DATA element.  How the data actually appears in the Data column depends on the format specification given in the Display field of the DATA Element dialog.  See the section Data Element Attributes in Template Files for more information about valid format specifications for various data types.

 

There are also columns for displaying the address of the field (in hex or decimal) and the size, which can also be useful for debugging a template.
