Basic Template Creation Tutorial

 

         

Welcome to the HexEdit Basic Template Creation Tutorial.

 

What are Templates?

 

Templates allow you to impose order on the chaos of the binary world.  All binary file formats have at least some sort of structure, which can be reflected in a template.  Many hex editors provide a "template" facility but HexEdit allows you to easily handle all sorts of situations that the others cannot such as:

  • optional parts whose presence depends other fields in the file

  • variable length arrays whose size depends on a count field or any expression

  • variable length arrays which are terminated by a flag field (or any boolean expression involving the fields of a record)

  • variable (even zero) length strings and blobs whose size depends on values of other fields or more complex expressions

  • non-sequential file processing such as a "header" stored at end of file

  • recursive data structures

Moreover, HexEdit allows you to validate the structure of a file.  Opening a badly formatted file with a template can allow you to instantly see problems that would take hours or even days to track down by analyzing the file by hand.

 

Of course, a template can never be as flexible as a custom program designed to display a format.  However, it typically only takes a few hours to create a template that would take weeks or months to create custom software to display.  Yet HexEdit templates are flexible enough to handle almost any data format.

 

Who Should Use this Tutorial?

 

This tutorial will be of use if you have never created a template before and want to learn how.  Or maybe you just want to find out what all the fuss is about templates — what they are and the sort of things they can do.

 

It should take less than 40 minutes to complete the tutorial.

 

Before You Start...

 

It is assumed that you have a basic understanding of the use of HexEdit and what binary files are for.

 

You can find out more about using templates in Using Templates, although it is not assumed you have read that.

 

What You will Learn

 

The tutorial will show you how to turn on template editing and create a simple template by modifying the "default" template.  The template you will create is for displaying the structure of normal text files.  It also shows how to save, test and edit a template.  To do this it demonstrates the use of some of the template editing dialogs.

 

The tutorial does not teach anything about the format of the underlying template (XML) files.  You do not need to know that to create and edit templates.  But to find out about the format of template files see Template Files.

 

There are five steps each with about five instructions.  The steps are:

 

1. Getting Started

2. Creating the Template

3. Saving the Template

4. Testing and Fixing

5. Finishing Touches

 

At the bottom of each step there is a "Next" button, which allows you to move to the next step (and a "Previous" button to allow you to go backwards).
