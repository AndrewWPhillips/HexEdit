Step 1. Getting Started

 

         

 

The first step is to turn on templates and template editing and open an example file.

 

Turn on Templates

 

Before you can modify templates you have to be able to view them.  This is done in the Display page of the Options dialog (accessed using the Tools/Options menu item) and selecting the "Use Templates" check box.  After that any file you open in HexEdit will display a template window.  The template window can be displayed in a "split" window next to the normal hex window or it can be displayed on top of/behind the hex window, using tabs to flip between them.

 

1. Open the Display Options Page and select the "Use Templates" and "Allow Editing checkboxes.  Don't change the other settings and click the OK button.

 



Figure 1.1 Turning On Template Editing in the Display Options page.

 

Open an Example File

 

To edit a template you need to have a file open.  The best file to use is an example of the type of file for which you are creating the template.  This allows you to get instant feedback on how your template works.  Of course, to exercise all the features of more complicated templates you need to test them with many example files.

 

In this case we are creating a simple template for text files so any text file will do.  These files often have an extension of ".TXT".

 

2. Open the file "RelNotes.txt" which is in the same folder that you installed HexEdit.

 

Default Template

 

After you open the file you should see the template window to the left of the normal hex window.  If there is no template associated with a file then the "default" template is used.  We are going to create a new template by modifying the "default" template.

 

At the top of the tree window there is a heading which should say "default" with a down-arrow to the right.  If the heading does not say "default" then some other template is associated with the file, so you need to click the down-arrow and select "default" from the list.

 

Underneath the heading is the tree itself which is used to view and edit the template.  At the moment this will just contain one line which has a yellow "Folder" icon with the name "Default".  This is the "root" of the tree and represents the whole file according to the template.  To the left of the "Folder" icon is a tree "+" symbol which allows you to expand the tree node.

 



Figure 1.2 RelNotes.txt using default template.

 

3. Expand the "Folder" node in the tree by clicking the "+" symbol.

 

You will see that the default template only contains a single data element with name "none".

 



Figure 1.3 After Expanding the Root Node of the Tree.

 

Displaying the Data Element Dialog

 

Most template changes are made through context menus that appear when you right click a node in the tree.

 

4. Right-click the "none" node and select the "Edit..." item at the bottom of the popup menu.

 



Figure 1.4 Editing the "none" node.

 

If you right-click and nothing happens then you do not have template editing enabled.  To turn on template editing make sure it is checked in the Display page of the Options dialog.  A quick way to turn it on is to right-click the "Default" (root) node and select "Allow Editing" from the popup menu.

 

You should now see the "Data Element" dialog which allows you to change the properties of the "none" data node.  Note that the name of the data element is "none", and the type of the element is also "none".  A type of "none" means that the data is zero or more bytes of no specific type.  This is often used to represent a BLOB (binary large object) or parts of the file which contain unknown or unspecified data.

 

In this case the "length" field is empty which means that this data field extends all the way to the end of file.  Hence the default template just contains one piece of data which takes up the whole file.  The other fields of the dialog will be explained below.

 



Figure 1.5 DATA Element dialog.

 

Click the "?" in the top-right of the dialog then click any control to get a brief tip on what it is for.  Click the "Help" button to get an overview of the purpose of the dialog and information on all the controls.

 

We will later delete the "none" element but for now just close the dialog.
