Bookmarks Dialog

 

 

This dialog allows you to manage your bookmarks.  Each bookmark has a name, that you give it, plus a file name and an address within that file.  (Note that a bookmark's position is automatically adjusted if you use HexEdit to insert or delete bytes before it in the file.)  Bookmarks also have two times: a modification time (when it was created or reassigned) and an access time (the last time you jumped to it).

 

Note that the dialog is modeless, so you can leave it on screen and it is available when needed.  In this way you can quickly add a new bookmark by typing a name and clicking Add or quickly jump to an existing one by double-clicking on it.  Like other modeless dialogs in HexEdit it also gets out of the way when you move the cursor position in the active file to a position where the dialog would hide it.

 

You can also resize the dialog if you want to see more or less of the bookmark list.

 

This dialog is invoked using the Edit Bookmarks command.

Name

To add a bookmark you first need to enter a name in the edit box.  Note that the edit box is unavailable if the active file has not yet been saved to disk (ie you created a new file but have not yet saved it).

Add

Click this button to add a new bookmark.  The name of the new bookmark is whatever you have typed into the Name edit box.  If a bookmark with that name already exists you are asked if you want to replace it.  The new bookmark is set to the current cursor position in the active file.

 

Note: You can't add a bookmark to a new file (ie one created with File/New) until it has been saved to disk (with File/Save or File/Save As).   Bookmarks must always be associated with a disk file name so cannot be added to a file until it has been saved.  The Add button is disabled (since no disk file exists) until you save the file.

List of bookmarks

Most of the dialog is taken up with the list which shows all of your bookmarks.  The list is split into (up to) 6 columns (described below).  You can show and hide any of the columns by right-clicking any column heading and selecting it in the list.  You can also resize any of the columns by dragging the separator (the vertical line between the column headings), or double-click the separator to size the column to the width of the widest item.

 

By default the bookmarks appear in the order they were added, but you can sort on any column by clicking on the column header.  Click the same column header again to reverse the order.

 

To select a bookmark, to delete or jump to, just click on its entry.  You can select more than one bookmark (to delete) by shift-clicking to select a contiguous group or by control-clicking to select or deselect individual bookmarks.  The columns are:

 

Column

Description

Name

The name that was given to the bookmark.  A name should not start with an underscore (_) nor contain a vertical bar (|).

File

This is the name of the file (excluding the full path, but including the extension) in which the bookmark is located.

Folder

This is the folder or path in which the file containing the bookmark is located.

Byte No

This is the address within the file of the bookmark.  If you insert or delete bytes earlier in the file using HexEdit then it is adjusted accordingly.

Modified

This is the date and time that the bookmark was created or last manually moved.  (Not changed when automatically moved by inserting/deleting.)

Accessed

This is the date and time that the bookmark was last jumped to.

 

Delete

Clicking this button immediately deletes all the selected bookmarks.  If there are no bookmarks selected in the list then this button is disabled.

Go To

Clicking this button jumps to the selected bookmark.  The file containing the bookmark is opened if necessary.  If there is no bookmark selected in the list or more than one bookmark is selected then this button is disabled.  You can also double-click a bookmark in the list to jump to it.  You can also easily jump to a bookmark in the current file using the Bookmark_tool.

Validate All

Use this button to check the validity of all bookmarks.  If the file for a bookmark no longer exists then the bookmark is deleted (with the proviso mentioned below under the Keep network and removeable option).  If a bookmark is past the end of a file then it is moved to the end of the file.

Keep network and removeable

When validating bookmarks you may not want to remove bookmarks for files that are not always available.  For example, you may have a bookmark for a file that exists on removeable media (floppies, zip disks, CDs, DVDs etc) or on a network drive that is not always available.  When this option is on and the Validate All option is used only bookmarks on fixed local drives will be deleted.
