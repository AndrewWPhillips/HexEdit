Step 3: Creating the Graphical Object Structure

 

In this step we will create the recursive structure for the graphical objects.

 

There are two ways to create structures in HexEdit templates:

  • 1. You can create a single instance of a structure whenever you need it using the STRUCT dialog.

  • 2. You can define a structure at the start of the template then use it throughout the template.

The first method is most often used.  It is useful for just grouping together related fields or for simple structures that occur in one or a few places.

 

The second method is useful where the same structure can occur in many places.  Defining the structure in one place can make a template smaller and faster.  It also makes the template easier to maintain as the structure only has to be changed in one place.

 

The second method is also necessary when recursive data structures are used.  Creating a recursive data structure is simply a matter of creating a structure definition that references itself.  This means that we can nest objects inside other objects to any arbitrary depth.

 

Structure Definition

 

We will create a single structure that handles all the variations of the graphical objects (rect, circle and group).

 

1. Right-click the root-node and select the "Add Struct Definition..." menu item.  In the dialog enter "object" for the type name.

 

All graphical objects start with the same two bytes - a "type" byte and a "flags" byte.

 

2. Click the "Insert" button and select "Data..." from the list.

3. Give the DATA Element the name "type" and make is a single-byte unsigned integer (ie, type="integer", format="unsigned", length="1").

4. Click the "Clone >>" button and make the new element's name "flags".  Leave the other fields the same but add "this == 0" to the "Domain" field.

 

The "flags" field is reserved for future use, and the specification says that it should currently always be zero, so we check this with the domain test.

 

Since the "type" field should only ever have a value of 0, 1, 2, or 10 we could also validate it with a domain expression.  However, a better way is to add an enum string.

 

5. Click the "<<" button to return to the "type" DATA Element.

6. Enter "{END, RECT, CIRCLE, GROUP=10}" in the "Domain" field..

 

An enum is like a C/C++ enum list and must be enclosed in braces {}.  Note that END, as the first in the list gets the value zero.  RECT and CIRCLE take the next sequential values, one and two respectively.  Since there is a gap to GROUP it is explicitly assigned.

 

 

Figure 1. DATA Element dialog showing enum.

 

7. Click OK to return to the "object" STRUCT dialog.

 

We handle the different graphical object types with three "IF" elements.  The order is not important but we will add the rect and circle objects first.

 

In HexEdit 3.0 an "IF" element can now have "ELSE" parts, which can be used to avoid requiring two "IF"s (with opposite boolean expressions) in succession.  This can make the template faster and is more convenient in many situations.  However, in this example we simply use three separate "IF"s without ELSE parts.

 

Rect Object

 

8. Select the blank Element at the bottom of the list in the STRUCT dialog, click "Insert" and select "If...".

9. In the IF dialog set "Condition" to "type == 1".  (A rect has a type of one.)

10. In the IF dialog click the "Insert" button and select "Struct..." and give it the name "rect".

 

Now we will add the fields for a rect which are all DATA elements

There should always at least one object in a group so we validate that "count" is greater than zero using the domain field.

 

Don't forget to clear the domain field for "x" and "y" after cloning the "count" DATA element.

 

Now we need to add the array of objects in the group.

 

20. In the "group" STRUCT dialog select the last "blank" element and insert a FOR (Insert/For...).

21. In the FOR dialog enter a name of "subobject".

 

Now we need to put the number of grouped objects into the FOR dialog "Count" field.  Hence a brief explanation on expressions is needed.

 

You can enter any expression in the "Count" field, but the result must be an integer (ie, not real etc).  You will also get an error if the value is negative but zero-sized arrays are permitted and often useful.

 

An expression can use the value of any field already encountered in the file.  When a variable name is used in an expression we need to find it based on its name.  First all sibling elements are checked then all parent siblings, grandparent siblings etc up to the top level of the tree.  Since you can use "." to access STRUCT members and "[expr]" to access array elements you can get the value of any data element so far encountered in the file.

 

22. In this case we just need to use the "count" DATA element which is a sibling of the FOR, so just enter "count" in the "Count" field.

 

You can find the name of any DATA element that you can use by clicking the arrow button to the right of the "Count" field.

 



Figure 4.  Selecting the "count" value in the FOR dialog.

 

Recursion

 

Finally we get to the recursive bit - we make the sub-element of the array an "object", ie the structure we are defining.

 

23. Click the "Insert" button in the FOR dialog and select "Use Defined Struct...".

24. In the USE STRUCT dialog leave the "Name" field blank and enter "object" into the "Type name" field.

 

You can create mutually recursive structures too.  (For example, struct A contains a struct B, and struct B in turn contains a struct A.)  You only need to decide when creating the first struct what the type name of the second struct will be and use that in the "Type name" field.

 

The sub-element of a FOR does not need a name (since they are accessed using the FOR name and an integer index) so we can leave the name blank.

 

25. Click OK to close the USE STRUCT dialog, then click OK to close the FOR dialog.

 



Figure 5.  Final "group" STRUCT dialog.

 

26. Click the OK in the three remaining dialogs to close: the "group" STRUCT dialog, the IF dialog and the "object" STRUCT dialog.

27. Now right-click the root node in the tree window and select "Expand All" to see the complete structure.

 



Figure 6.  Complete "object" structure definition in the tree.
