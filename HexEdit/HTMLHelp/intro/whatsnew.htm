• Edit any volume or raw disk device (raw disk editing not supported for 9X)

  • Open Disk dialog shows all available devices and volumes

    • Full info. is shown so you know you are using the correct device

  • Edit volumes under NT/2000/XP

    • edit any mounted or unmounted NTFS or FAT volume

  • Edit raw disks under NT/2000/XP

    • edit hard disks, floppy disks, removeable disks, flash disks etc

    • view CD, DVD disks

    • edit disks and partitions of other OS (Linux etc)

    • unlike other disk editors the real device size is correctly detected

  • Edit volumes under Windows 9X/ME

    • all FAT filesystems supported

  • Example template for floppy (FAT12) volumes

    • recursive directory structure allows drilling down to any directory in tree view

  • Support for bit-fields

    • different underlying storage unit sizes supported (8, 16, 32 or 64-bit)

    • different fill directions supported (top to bottom or bottom to top)

    • arrays of bit-fields permitted

    • C/C++ parsing (see below) supports bit-fields

    • Display and editing of bit-field values (Data column in template view)

    • enums allowed with bit-fields too (displays drop list in Data column)

  • Multi-way branch (switch/case) is now supported

    • most requested template feature

    • convenient editing using CASE dialog (similar to STRUCT dialog)

    • each case can match a single integer constant or one or more ranges

    • "default" case is handled by adding a case with full range (at end of list)

    • only the matching case is shown in the template view (unless in edit mode)

    • if no cases match then nothing is shown in template view

  • Parsing of data types from C/C++ source code

    • will parse almost any C/C++ code

    • struct, union, class, enum, bit-field etc are all parsed correctly

    • handles all basic type including arrays and pointers

    • constant (compile time) expression evaluation (eg for array size)

    • supports macros (#define) with and without parameters

    • supports #include to any depth

    • supports conditional compilation (#ifdef etc)

    • ability to save parsed constants (enum elements and static const)

    • ability to save parsed types (enum, struct declaration, typedef etc)

    • full member alignment (padding) support including #pragma pack

    • padding fields are unnamed - not shown in template view (except in edit mode)

    • pregenerated standard types (for VC++) to save searching for headers

    • all types can be completely customized to any C/C++ compiler

      • size of all integer types (char, short, int, long)

      • size of floating point types (float, double)

      • endianness

      • size of enum, pointer, function pointer etc

      • direction and underlying storage unit for bit-fields

      • 8 different time_t formats supported

      • all other standard types (struct tm, lconv, etc)

      • easily add non-standard types (eg, long long, struct _complex ...)

    • complete pregenerated set of (218) types from Windows headers

    • see C/C++ Parser for Templates for more information

  • Many new functions allowed in expressions

    • 36 functions for integer, string, date, floating point etc

  • New example templates including floppy (FAT12) for use with disk editor

  • Ability to hide elements from the user of the template (unless in edit mode)

    • Elements without a name are not displayed

    • Elements that are not present are not displayed

      • empty FOR (ie zero-sized array)

      • IF and CASE elements are not shown (only sub-element if present)

      • IF element and sub-element not shown if expression is false

      • CASE and sub-elements not shown if no case matches the expression

      • unmatched sub-elements of CASE and IF-ELSE are not shown

  • New MSI based installer

    • faster, simpler, and more reliable installation

    • support for new options such as repair

    • uninstall removes all files and registry settings

  • Sounds for alert and error conditions can be changed

    • Illegal characters, background search finished etc

    • Configure sounds using Control Panel Sounds page

  • New "Insert Block" and "New File" options

  • Email dialog now allows attachment of any file

    • default is to attach current file being edited

    • quickly send file to HexEdit support for assistance

  • Visualizations (skins) now work correctly

    • Additional skins can be downloaded

What’s New in 3.1

 

Version 3.1 adds the new feature of disk editing as well as major improvements to templates, such as bit-fields, multi-way branches (switch/case) and extraction of types from C/C++ source code.  There are also other several other improvements and some minor bug fixes.
