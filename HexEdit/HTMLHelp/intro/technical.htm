Technical Details

 

Bugs

 

For large files the print preview does not display the correct page numbers, and will only display up to the first 32767 pages.  A workaround is to select the part of the file to be printed and use the "selection" option in the Print dialog.

 

At the time of release there are no other known bugs.  For the latest bug information go to the Bug Forum on our Website.

 

Version Numbers

 

HexEdit version numbers consist of 4 parts: major version, minor version + variation, update release and revision.  Internally these are stored as 4 numbers.

 

Major version: A new major version number indicates a major new release (though minor releases are not insignificant)

Minor version: Each new release increases the minor version number.  A more significant release may jump to a 5 minor version or the next major version.

Variation: An update letter may be added after the minor version to indicate a variation on a release. This could be for minor bug fixes or small changes.

Update release: If a major bug is found after release with no workaround then an update release will be created.

Revision: This is an internal number indicating the SVN revision number.  It is different for every release and monotonically increasing over time.

 

There have been 3 major version and several minor versions and two variation (the free versions 2.5F and 3.0F).  It has never been necessary to produce an update release.

 

Internally the original minor version is stored as a single digit, but variations multiply by 10 and add 1 to 8 for variations "A" to "H".

 

Note: The file time (modified time) of HexEdit.exe and other files are modified for a release to indicate the version number.  For example for 2.5F the time is 2:56AM.

Limitations

 

There are very few limitations in the use of HexEdit 5.0.  Limitations in earlier releases have been removed.  For example, some operations (encryption, conversions, compression etc) were limited to selections of 4 Gigabytes, but now can work on any sized selections.

 

There is no limit on the number of undo operations or the size of keystroke macros.

 

Old versions of HexEdit had limitations on the amount of data that could be placed onto the clipboard due to limitations of Windows. HexEdit 4.0 and later avoid this problem by use of a custom clipboard format that makes use of a temporary file and only uses a small amount of memory.

 

The maximum line length (ie, number of columns) is 32767.

 

International Support

 

Unfortunately HexEdit only currently supports English in its menus, messages, help etc. The documentation is now in US English as more users are familiar with that than other varieties of English.

 

Luckily HexEdit does support international numeric and date formats when these are correctly configured in the Windows control panel.

 

Development Environment

 

HexEdit 1.0 was developed using Microsoft Visual C++ 5.0 and MFC. The system was NT 4.0 SR3 on a Pentium 100 system with 80 Mbytes of RAM and two hard disks (EIDE and SCSI).

 

HexEdit versions 1.1 to 2.2 were developed using Microsoft Visual C++ 6.0 under Windows 98 on a Celeron at 487.5 MHz with 128 Mbytes RAM and two monitors (19” and 15”).

 

HexEdit 2.5 to 3.1 were developed using VC++ 6.0, Windows XP on an Athlon 1600 (K7S5A mobo) with 512 Mbytes RAM and 2 monitors (19" @1600x1200 GForce 4) (17" @1024x768 ET6000).

 

HexEdit 3.2 to 3.6 were developed Visual Studio 2003 (MFC 7.1) on XP SP2 with Sempron 2300, 1.5 Gbytes RAM.  (This is dual boot with Vista Ultimate but I never use Vista except for testing HexEdit on it.)

 

HexEdit 4.0 was developed using Visual Studio 2008, VS2010 and VS2012.  I use a dual boot XP/Win7 system but test on XP, Vista. Windows 7 and Windows 8 using VMware images.

 

HexEdit 5.0 was developed using Visual Studio 2010, VS2012 and VS2013 under Windows 7.
