Quick Start

 

Here is some quick information on some useful features that are not immediately obvious.  If you are not familiar with using Windows programs and binary file editor in particular you then you should first read the Overview section and possibly the All About Binary Files section.

 

You may prefer to learn how to use HexEdit using our video tutorials - see http://www.hexeditpro.com/videos.aspx.

 

Command Help: You can get an idea of a lot of the features by trying all the commands on the menus and the four toolbars.  If you can't work out the purpose for a command you can hold the mouse over the menu item or toolbar button and press F1.  You can also get help on buttons and other things by pressing Shift+F1 and clicking on it.

 

Dialog Help: Right click a control in a dialog to get help on it.  Press F1 or click the Help button get more help on the whole dialog.  See help for more information.

 

Read Only Files: You may not be able to open some files for modification, for example if in use or you do not have permissions.

 

Read Only Windows: You can have several windows open on the same file, some or all of which can be read-only even if the file is not open read-only.  To toggle between read-only and read-write modes just double-click the RO/RW pane in the status bar.

 

Shareable Files: You can open files in shareable mode (see File Open dialog) if you want to allow other software to modify the file while you are viewing or editing it.  However, this may be confusing as HexEdit does not read the whole file into memory but simply updates its display directly from the disk file.

 

Mark: Marking an address (eg by double-clicking on a byte) has numerous uses.  For example, the status bar can show the distance to the mark in hex and decimal.

 

Bookmarks: Quickly add/remove a bookmark using Ctrl+F2 or use the Bookmarks window to add a named bookmark.  Use F2 and Shift+F2 to jump to the next/previous bookmark in a file.

 

Highlighter: The highlighter is useful for remembering one or more ranges of bytes.  You can set the color of the highlighter in the Color Options page.

 

Standard Toolbar: The Standard Toolbar has the Autofit button , Character Set drop-list and other buttons for changing a window's display. See Window Display Options page for more display options.

 

Color Scheme Tool: The Color Scheme Tool allows you to quickly switch between color schemes.  Custom color schemes can be set up in the Color Options page of the Options dialog.  Among other things color schemes allow you to set byte ranges to be displayed in different colors to easily spot things in the file.

 

Jump Tools: The Edit Bar contains the Hex and Decimal Address Tools . that allow you to quickly enter and jump to an address.  As you move through a file they also update to show the current address.  Moreover you can enter any integer expression (including using bookmarks and assigning to variables).  Finally, they can be used for base conversion as typing a number into one, automatically updates the other.

 

Find Tool: The Find Tool , allows you to search for a sequence of bytes. Just type some hex digits and press Enter.  To do a string search type “=” at the start. For a case-insensitive string search use a tilde (~) instead.

 

Format bar: The Format Bar has buttons for changing the display including tools for changing the font and font size .

 

Bookmarks tool: The Navigation Bar has the bookmarks tool which allows you to quickly jump to any bookmark in the current file.  Moreover, as you move around it updates to show the closest bookmark above the current location.

 

Ruler: The ruler is shown at the top of each window.  Different rulers are shown depending on whether addresses are shown in hex or decimal.  You can drag the handles in the ruler to change the format of the display such as the number of columns.  See Workspace Display for ruler options.

 

Status Bar: The status bar has several panes that display useful information.  Hold the mouse over a pane to see a tip window about what it is for.  Double-clicking some panes toggles a mode - eg double-click the REC pane to record a macro.  Right-clicking most panes displays a popup menu of related commands.

Properties: The properties dialog allows you to view information about the active file (see File Properties Page).  It is mainly used for showing information about the byte at the cursor (and following bytes).  There are a large number of character (Byte Properties Page), integer (Integer Values Page), floating point (Real Values Page), date (Date/Time Values Page) etc formats available.

  • As you move around the file the current value is updated to reflect the current byte(s)

  • You can change most values just by editing the field in the dialog (hit Esc to abandon changes or Enter to accept them)

  • Multi-byte fields can be big-endian or little-endian

  • Byte Properties Page allows viewing characters in Unicode or any code page supported by Windows

  • You can double-click any bit of the current byte in the Byte Properties page to toggle it

  • Integer Values Page supports many formats even less common ones like Sign and Magnitude

  • Real Values Page for IEEE and IBM floating point formats, Delphi real48, C# Decimal, etc

  • Date/Time Values Page supports many date formats including several variations of time_t

Category, Comments, Keywords: Also in the properties dialog you can enter a category etc for the active file in the Summary Page.  This information is saved and is visible in many places such as the Recently Opened Files Dialog and in the HexEdit Explorer window.

 

Recent File List: The Recently Opened Files Dialog provides a list of all files you have opened.  You can sort the list by category, or by the date/time the file was last opened in HexEdit, or any column.  You can open file(s) from the list and remove files from the list, etc.

 

Window Formatting: How the file appears in the window (row size, colors etc) is remembered between sessions in a special file called "RecentFileList".  If you remove a file from the abovementioned Recently Opened Files Dialog then this information is lost.

 

Background searches: Whenever you perform a search using the Find Tool or the Find Dialog a background search is started to search the rest of the file (and all other open files).  This greatly speeds later searches for the same thing.  The number of found occurrences is displayed in the search pane of the status bar (or it shows the progress if the background search has not finished).

 

Wildcard searches: The Find Dialog's Find Text page supports a wild card character. The Find Hex page supports masking down to the level of individual bits.

 

Aligned searches: The Find Hex page and Find Number page support full alignment and offset searches.  For example, you could just search for a specific field in a file containing many fixed length records.

 

Views: The default view of a file is the Hex view but you can also open other views of the same files (such as Template and Aerial views).  These views can be shown in a "split" display (where you can see both views at once) or in a "tabbed" display (where you switch between them using the tabs at the bottom of the window).

 

Aerial View: The Aerial View lets you get a high-level overview of a file.  Each byte of the file is represented by one pixel (though you can zoom in).  It is important to use an appropriate color scheme since that determines the pixel colors.  If you know nothing of the file format just select the "Rainbow" scheme from the Color Scheme Tool.

 

Aerial View Zoom: When you zoom using Ctrl+Mouse Wheel in the Aerial View the mouse pointer is moved automatically so it always remains over the same byte. Double-clicking a pixel in the Aerial View selects the byte in the main window (hex view).

 

Templates: Templates are an extremely powerful feature that lets you intelligently display information from a file including repeated, optional and out-of-sequence data.  They are flexible enough to handle almost any data format including recursive data structures.  See Using Templates, Editing templates, Advanced Template Tutorial.

 

C/C++ Parser: You can parse C/C++ source code to generate a template or part of a template.  This is typically used to extract a struct from a C header file but can handle just about any C and C++ source code thrown at it.  See C/C++ Parser for Templates.

 

Template Files: Templates are stored as XML files.  Generally it is easier to use the built-in template editing but sometimes it is useful to edit the XML files directly. See Template Files for more details.

 

Expressions: HexEdit includes the ability to evaluate expressions (using the syntax and precedence of expressions in C/C++).  These are used extensively in templates but can also be used in other places like the calculator and jump tools.  See Expressions.

 

Calculator: The calculator allows you to do any sort of integer calculation and display the result in any radix or to any precision.  It is often used to calculate an address that can be jumped to using the GO button.  The file access buttons allow you to manipulate values from the currently active file, which is particularly useful when used in keystroke macros.

 

Macros: Keystroke macros allow you to repeat a repetitive task by recording it once and playing it back many times. Not only “keystrokes” are recorded but menu items, mouse selections and changing window focus etc.

 

Macro files: Macros can be saved to file (extension .HEM) and reloaded at a later time. If a macro is called “autoexec” it is run whenever HexEdit is started.  If a macro is called "_xyz" (ie a file extension preceded by an underscore) it is run whenever a file with the extension "xyz" is opened.

 

Undo: Everything can be undone (not just file changes) including cursor movements, formatting changes etc.  There is no redo.

 

Change Tracking: Allows you to see what bytes have been modified.  If INS mode is on you can also see bytes that have been inserted and where they have been deleted.

 

Selection Length: Hold down the Shift key at any time to see the selection length in a tip window.  Some commands also require the selection to be a multiple of a factor of two (eg, 2, 4, or 8) so the tip window also displays the largest power of 2 by which the length is divisible.

 

Clipboard: Cutting, pasting etc stores data on the clipboard in both binary and text formats.  This allows working with text data (eg to copy to a text editor) or binary data (eg to use with Visual Studio binary editor).

 

Copy as C Source: You can copy data to the clipboard converting to a large number of text formats (also see Copy As Hex Text).  The ability to copy as C source creates text suitable for initializing an array in C, C++ and other languages.  It can also create CSV integer or real data for pasting into a spreadsheet.

 

Highlighter: The highlighter can also be used as a multi-part (ie disconnected) selection -- for example in Export Motorola S1 Records.

 

Info Tips: When you hover the mouse over a byte a fully customizable tip window can be shown. It can show any number of tips with complete control over the display format.  Simple values such as the address or the current data (as integer, real, date etc) can be shown or used in an expression.  See Info Tips Page.

 

Customization: You can fully customize the toolbars & main menu (see Customize Toolbars page) and keyboard shortcuts (Customize Keyboard page).  You can also customize many context (popup) menus which are shown when you right click on various things and various double-click events (Customize Mouse page).

 

Generating data: The Insert Block command allows you to generate data in many different formats, using the Insert Block Dialog.  The same options are available in the New File command.

 

ZLIB: All options of the industry standard ZLIB algorithm are supported.  The Compression Settings Dialog allows you to experiment with the settings without writing any code.

 

Large operations: All operations on the selection, such as compression, encryption, export etc are supported up to any size.  Of course, for a selection of hundreds of GigaBytes any operation will take a long time.

 

Long Operations: Operations that take a long time can be aborted by pressing the space bar.

 

Export/Import: You can currently import and export in Motorola-S and Intel-hex formats. All options are supported including discontiguous records (using the Highlighter).

 

Opening Files: There are many ways to quickly open files

  • drag file(s) from Windows Explorer and drop onto the HexEdit window

  • right-click on the file in Windows Explorer and select Open with HexEdit

  • specify the files on the command line including using wild-cards to quickly open many similar files

  • double-click the file in HexEdit Explorer

  • select and open one or more files from the Recently Opened Files Dialog

File Filter List: You can create your own list of file filters depending on the types of files you work with.  The filter list is presented in all File Open/Save dialogs as well as the Explorer Dialog.

 

File Management: HexEdit Explorer allows you to do similar things to Windows Explorer but you can also:

  • quickly open the file in HexEdit

  • view columns showing category, keywords, comments

  • sort by date/time a file was last opened in HexEdit

  • filter the list of files based on wildcards or the file filters list

  • easily show/hide files with the hidden attribute

Shortcut Keys: Many commands have shortcut keys (and you can assign your own - see Customize Keyboard).  When you hover the mouse over a menu item or toolbar button a tip window is displayed which shows shortcut keys if any.  The following keys are not associated with any menu item or toolbar button:

 

Ctrl+L

Centre cursor within the window, and redraw the window

Ctrl+U

Scroll the window up

Ctrl+D

Scroll the window down

Tab

Swap between hex and character areas

Alt+Enter

Change the offset so the current column is at the left of display

Ctrl+F3

Search forward for next occurrence of currently selected bytes
