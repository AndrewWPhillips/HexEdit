SHA1 command

 

This performs a cryptographic checksum (also called a message digest or one-way hash function) on the currently selected bytes using the SHA1 algorithm.  The result is a 160-bit integer which is placed in the Calculator and (optionally) displayed as hex (as is conventional) in a popup window.  The value is grouped into 8 groups each of 4 hex digits for ease of reading.

 

To copy the contents of the SHA1 results dialog just press Ctrl+C when it is shown.

 

It is currently believed to be computationally infeasible for someone to calculate a set of bytes that would generate a specified SHA1 value.  In other words if you know the SHA1 value for a set of bytes you can be sure they have not been tampered with by checking that they still generate the same value.  (This contrasts with CRCs which, although excellent for detecting accidental changes to data, are no guarantee that the data has not been deliberately tampered with.)

 

Like MD5, a SHA1 value is often used to verify that files downloaded from the Internet have not been modified.  However, SHA1 is considered more secure since it has more bits and there are also some doubts about the veracity of the MD5 algorithm. However, even better are SHA2 and SHA3 digests which are now supported by HexEdit.

 

To generate the value for the whole file using HexEdit just select the whole file (eg Ctrl+A) before using this command.
