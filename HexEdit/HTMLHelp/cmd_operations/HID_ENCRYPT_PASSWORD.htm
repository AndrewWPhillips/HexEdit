Encryption Password (Operations menu)

 

This command lets you set the password using the Encryption Password Dialog that is then used as the "key" in subsequent uses of the Encrypt and Decrypt commands.  See Encryption for more information.

 

Note that if no password has been entered when the Encrypt or Decrypt command is invoked then a password is automatically asked for.  This is useful in keystroke macros - see also the Clear Password command.

 

If it is important that the data is very secure then choosing a good password is important.  For this reason the default minimum password length is 8 characters, but you can reduce this if you don't require this level of security.  As you are probably aware a good password is not a common word or phrase and probably should include some punctuation characters.

 

Be careful not to lose your password as any data encrypted using it can probably only be recovered by decrypting with the same password.  Remember that passwords are case-sensitive, so "Banana" is not the same as "banana".  Also if you record typing in a password in a keystroke macro and save the macro to disk the password is stored as plain text in the macro file, for anyone to see, hence it is recommended this is not done (see the Clear Password command).
