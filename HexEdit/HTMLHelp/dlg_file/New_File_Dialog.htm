New File dialog

 

 

 

This dialog appears when you invoke the File/New command.  It allows you to create a file of any size filled with various values, which can then be edited as normal.

 

You can create very large files, but they may take some time to generate.  The filesystem may limit file sizes (eg FAT has a limit of 4 GBytes).  Also for large files HexEdit may need to create a temporary disk file, but if the "Prompt for disk name now" option is used the data is written to directly to the final destination file to save copying later.

 

Size

Several options are available for specifying the initial file size.  If the size to be used is greater than the length of the data the bytes of the data are repeated up to the desired length.  Files up to 1 Terabyte in length can be created.

 

Data

The file size is taken from the length of the selected data (see Fill With options below) -- for example, the length of the entered string or the length of the data on the clipboard -- and multiplied by the "by:" factor to the right.

 

by:

The file length is the length of the data multiplied by this factor.  The actual size is displayed to the right, after the equals sign (=).

 

Take Size from Calculator

The size of the new file is the value currently shown in the calculator.  If the value is negative or zero an empty file is created.  This option can be useful in keystroke macros.

 

Specified

The size is specified explicitly as either hex or decimal.

 

Decimal:

Enter the size of the new file you want in decimal.

 

Hex:

Enter the size you want in hex.

 

Prompt for Disk File Name

If you select this option the file is immediately created on disk.  After you click OK you are prompted for a disk file name.  This can save time later when the file is eventually saved if the file is very large.

 

Fill With

Choose how the values used to fill the new file are created by selecting from the radio buttons described below.  The bytes are repeated until a file of the size specified is created.

 

Hex

Enter one or more byte values in hex.  To make a file just containing null bytes simply use "00".

 

String

Enter one or more characters.  The values are taken from the character set specified below.

 

Char Set

The character set used to generate byte values from the string.  You can create ASCII, EBCDIC or Unicode data.

 

Clipboard

The data is taken from the clipboard (binary data or text if no binary data is available).  A preview of the data on the clipboard is shown (as hex for binary data or as a string for text).  If no valid formats are on the clipboard this option is disabled.

 

Length

Displays the number of bytes currently on the clipboard.

 

Random

Generates random byte values.  You can enter one or more ranges.  For example, use "65-90, 97-122" to generate random uppercase and lowercase ASCII letters.  Use the range "0-255" to generate all possible byte values.  The PRNG algorithm is seeded by the Random Seed command or the Seed Random Number Generator from Calculator command.

 

Note that you can use this to generate random integers too (of any size).  However, you cannot use it to generate random floating point values - for one, you will generate some invalid floating point values (NaNs) - see Floating Point Numbers for more information.  (A facility to generate random floating point number may be added in the future.)

 

The algorithm used quickly generates adequate random numbers.  For a high quality PRNG use the "Operations/Random Fill/Generate Good" menu item.

 

Number

Enter a number that is used to fill the file up to the size entered (above).  To use IEEE floating point numbers enter a number with a decimal point (such as "1.0") and/or an exponent (such as "-1e30").

 

Type

The type of integer or floating point number.  You can use 1, 2, 4 or 8 byte integers, or 4 or 8 byte IEEE floating point numbers.

 

Big-endian

Determines the byte order of the number entered.  (See the information on byte order in Numeric Operations.)
