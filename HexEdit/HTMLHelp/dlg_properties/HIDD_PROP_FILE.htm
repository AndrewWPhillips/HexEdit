File Properties Page

 

 

This page shows information from the disk file associated with the active window. If the file has not yet been saved most of these fields will be blank. Note that the Size field has been moved to the Summary Page.

 

Name

The disk file name of the file in the active window.

Type

The type of the file according to the system registry is displayed to the right of the name. Nothing is displayed if the filename has no extension.

Location

The complete path of the folder (directory) where the file is stored.

 

The file size (both in memory and on disk) is now shown in the Summary page.

 

Created

This is the date and time that the file was created according to the operating system (actually the filesystem).  Note that some files under the FAT filesystem may not have a stored Created and Accessed time, in which case the Modified time is reported for these times too.

 

When you copy a file its modified time is the same as the original but its creation time is the time it was copied -- this results in the anomaly that the file appears to have been last modified before it was created.

Modified

This is the modified time of the disk file. If you have saved the file since opening it within HexEdit Pro then this is the time that you last saved it. If you have not saved the file since opening it, this is the time that it was most recently modified before then.

Accessed

This is the date and time that the file was last accessed.  As you are currently viewing the file this should be a time in the very recent past.

 

Note that for some files this may not be the time of last access but actually the time of last modification (see the note on the Created time above).

Read only

Is the file marked read only?

Hidden

Is the file marked as hidden?

System

Is the file marked as a system file?

Archived

This shows the status of the files Archive flag.  This flag is used by some backup software to indicate whether a file has been backed up since it has been changed.  The backup software will set the flag when the file has been backed up (or restored); the flag is cleared whenever the file is modified.
