How Do I Convert EBCDIC text to ASCII?

 

As there is not an exact match between the EBCDIC and ASCII character sets converting between them can be done in several ways.  Three different options are:

 

1. The simplest way for simple text is to just turn on EBCDIC mode using the EBCDIC Characters command, select the text and copy it to the clipboard using the Copy command.  You can then paste it into another program (like Word or NotePad) as normal text.  With this method any EBCDIC control characters and characters that cannot be converted to ASCII are simply thrown away.

 

If you then want to paste back in HexEdit you must use Paste As ASCII.  If you just use the Paste command then HexEdit will paste exactly the same bytes back into the file.

 

The technical reason for this is that whenever you cut or copy HexEdit places binary data and text data on the clipboard.  The binary data is necessary so that copying and pasting of all byte values works, since not all byte values (such as null bytes) can be placed on the clipboard as text.  When you paste in HexEdit it looks for this binary data before looking for text data.  (Note that this binary data format is the same as that used when copying to the clipboard in the Developer Studio hex editor.)

 

2. To convert a block of bytes in place in a file use the Convert EBCDIC to ASCII command on the Operations menu.  This has the advantage that the selection length is not changed.  However, any bytes that cannot be converted are replaced with a null byte (ie, a byte with value zero).

 

3. A simple way to find the ASCII values corresponding to a small number of EBCDIC values is to use the Find Hex page and the Find Text page of the Find dialog.  Select the EBCDIC character set radio button on the Text Find page then on the Hex page type the hex value of the EBCDIC text, go back to the Text Find page and select ASCII mode, then select the Hex Find page again.  The find box now shows the hex values of the corresponding ASCII text.

 

Custom Conversions

 

As there is not a simple one-to-one correspondence between ASCII and EBCDIC the way conversions are done is fairly subjective.  Some characters may be changed to similar looking characters and some characters have to be ignored completely.

 

If you are not happy with the way HexEdit does this conversion then you can modify the file called EBCDIC.TAB which is kept in the same directory as HexEdit.EXE.  This file contains 256 bytes that provide a lookup table, that given an EBCDIC value as the index stores the corresponding ASCII value.  This file is also used to generate the inverse table for ASCII to EBCDIC conversion.

 

Note that modifying this file affects how all the above conversions from EBCDIC to ASCII and from ASCII to EBCDIC are done.  It also affects how EBCDIC text is displayed and how text searches in EBCDIC mode are done.
