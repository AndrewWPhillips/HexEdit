How Do I Enter a Number?

 

You can change numbers in binary files in many ways in HexEdit .  First you have to decide what sort of number it is.  That is how many bits (8, 16, 32 or 64), byte-order (big- or little-endian), and format (unsigned, 2's complement signed etc).  If the number is a floating pointing value see How Do I Enter a Floating Point Number?

 

If the file containing the number was created on a Windows 95/98/NT etc then it will probably be a little-endian 32 bit value.  But very small numbers may be stored in single byte.  Files created by MSDOS/16 bit Windows often use 16 bit numbers, but may use 32 bit numbers for large values and 8 bit numbers for small values.  See Numbers for more information about number formats.

 

To enter a number in decimal, open the Properties dialog (using the Display Properties Command) and select the Decimal Values Page, then move the cursor to the first byte of the number in the file.  Select the appropriate byte order and format in the dialog.  If you know the value that the file should already contain (ie, the value to be replaced) then you can check it now to make sure you have the correct file location and format.

 

Now type the new number into one of the four Values fields (depending on whether the number is 1, 2, 4 or 8 bytes in length).  Press Enter to accept the number you have entered, or press Escape to abort the change.

 

Note that the Values fields may be disabled if the file is read-only or you are at the end of the file.  If you want to append a number to the file you will have to add some extra bytes to the file (in INS mode) before using the above method to change them.

 

You can also enter an 8 bit (single byte) value as octal or binary in the Character Properties Page.  To enter a number in hex just edit the hex values in the hex area of the file display.
