How Do I Search for a Floating Point Number?

 

Even if you know that you are searching for a floating point number you still need to know if it is a 32-bit or 64-bit number (and the byte order too if it comes from a non-Intel environment).  If you are not sure you can search for both formats.  See Floating Point Numbers for more information about floating point number formats.

 

Keep in mind that a number with decimal places may actually be a "fixed-point" number and is actually stored as an integral value (see How Do I Search for a Number? for how to search for integers).  For example, currency values (such as dollars and cents) may be displayed with a decimal point but are stored as integers (such as the number of cents or hundredths of cents).  Also the number may just be stored as text.

 

Once you have decided what format the number is in you can search for it using the Number page of the Find dialog.

 

Important

 

There is a significant problem with searching for floating point numbers that you should be aware of.  Although two floating point numbers may be displayed as the same value they may not necessarily compare equal.  Even though a number may be displayed as 10 it may be stored internally as a number more like 9.9999999999 or 10.0000000002.  This often occurs for numbers which are calculated (eg. 30/3 is rarely exactly 10) or for numbers that cannot be represented exactly in a binary system (eg. 1/10).

 

This means that if you search for the exact representation of 10 you won't find it if the number is really stored as 9.9999999999.
