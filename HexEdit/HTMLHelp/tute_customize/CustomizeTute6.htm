Step 6. Info Tips

 

         

 

Many programs provide useful information in a pop-up "tip" window when you hold the mouse over an object of interest.  HexEdit 3.2 introduced a very useful new feature that provides this convenience but goes further -- it allows you to display an unlimited number of completely customizable tips.  We will demonstrate a few of the limitless possibilities.

 

Adding a Tip

 

First we need a test file to work with.  A useful type of file is one that has bytes with all possible bit patterns.  HexEdit provides a macro to create such a file which we will run now.

 

1. Open HexEdit and select Play from the Tools menu (Tools/Play...).

2. Select the "AllChars" macro in the list and click OK.

 

 

3. Click OK to close the dialog that explains the purpose of the macro.

4. Click OK if you are asked to turn off read-only mode and/or turn on insert mode.

 

After a few seconds you will see a file of length 256 containing bytes with all the possible bit patterns that can be stored in a byte.  You can save this file for later use, or just regenerate it whenever you want.

 

5. Turn off change tracking (red background) if you find it distracting.

 

Imagine that you need to inspect the exact bit pattern of 64-bit integers throughout a file.

 

6. Open the Options dialog (Tools/Options...) and select the "Info Tips" tab.

7. Click the "new" button (tip = New Info Tip) near the top of the dialog.

 

A new tip is created underneath the Bookmarks tip.

 

The Bookmarks tip is a special tip that cannot be deleted and is always at the top of the list.  It shows the name of a bookmark but only if the mouse is over a bookmark.

 

The new tip is enabled by default but you need to fill in the other columns.

 

8. In the Name column (next to the checkbox) enter "Quad word bits".

9. In the "Expression" column click the down-arrow button and select "uqword" from the list.

10. In the "Format" column enter "binary" or "bin" or select it from the list.

 

 

11. Drag the Transparency slider (at the top) towards the right and click OK.

12. Hold the mouse over the first byte of the file we created above to see the tip.

 

 

 

If you have the tips set to a low transparency you can make the tip fully opaque by moving the mouse over it.

 

 

Byte Order

 

In the above binary tip it is important to understand where each bit comes from.  The identifier "uqword" is an abbreviation for unsigned quad word, which is an 8-byte integer, ie, 64 bits.  This uses the byte under the mouse pointer and the next seven bytes in the file.  When displayed in binary, these are grouped into 8 groups of 8 bits each, where each group corresponds to a byte.

 

The default byte order is little-endian in HexEdit as that is what Intel processors (on which Windows is based) use.  This means that the bits for the left-most byte (the byte under the mouse cursor) are the least significant and shown at the right end of the tip.

 

The byte order used for tips is taken from the byte order setting for the active file.  This is shown in the status bar BE/LE pane.  It's also seen as the Big-Endian checkbox in the Windows page of the Options dialog and also in the "Big-Endian" checkbox in the calculator.

 

13. Double-click the "LE" pane in the status bar so it changes to "BE" to change to big-endian format.

 

This changes to big-endian format for this file only.  The setting is remembered the next time you open the file.

 

14. Hold the mouse over the first byte of the file to see the tip with the big-endian byte order.

 

 

Byte order also affects other types not just integers, including floating point, date, and even the order of bytes in Unicode characters.  

 

Another thing to remember is that if there are not enough bytes to the end of file for the tip then it is displayed as blank.

 

15. Hold the mouse over any of the last seven bytes of the file.  The quad words tip is blank.

 

 

Types

 

As you might have guessed the type of an info tip can be other types of values apart from integers. It can also be a floating-point value, a date, a string or a boolean.  See the topic on HexEdit Expressions for more details.

 

We will experiment with a date field, but we first need to open a file containing dates - luckily the tutorial.HEG file you saw in the previous step has two date fields near the start of the file.

 

16. Open tutorial.HEG again.  (It can be found in the directory where HexEdit is installed.)

17. Open the Options dialog and select the "Info Tips" tab.

18. Click the new tip button near the top of the dialog.

19. In the "Name" column (next to the checkbox) enter "Test date".

20. In the "Expression" column click the arrow button and select "time_t" from the list.

21. In the "Format" column enter "%c" or select it from the list.

 

 

22. Click OK and verify that dates are being display correctly when you hold the mouse over the 4th and 8th bytes of the file.

 

The date fields in the tutorial.HEG file are at the 4th and 8th bytes (ie addresses 3 and 7) and contain the time_t dates near the start of 1990 and 1994 respectively.

 

The exact values displayed for the dates depend on your local time zone as time_t values are stored as UTC (universal coordinated time).  For GMT (Zulu) time this will be midnight.

 

23. Open the Options dialog again.

24. Change the format for the "Test date" tip to be "%A".

25. Click OK and check the tip by hovering over the date fields again.

 

The "%A" format specification is for the day of the week.  For the above date fields this should display "Sunday" and "Friday" or "Monday" and "Saturday", depending on your time zone.  See "strftime" in any C-language reference manual for details on all the available format specs for dates.

 

 

Predefined values

 

The names you see in the "Expressions" drop-down list are all the predefined values that can be used in the expression for the tip.  Most of these relate to the bytes at the mouse position in various formats - ie, as integers, reals, or dates in various sizes and formats.

 

Most of the other predefined values are for addresses in the file.  For example, "address" is the address in the file of the byte the mouse is held over, "cursor" is the address of the flashing insertion point in the file (also called the caret), "eof" is the address of the end of file (ie the length of the file), "mark" is the address of the mark etc.

 

We will now add a tip that displays the distance to the end of file.

 

26. Open the Options dialog and select the "Info Tips" tab.

27. Click the new tip button near the top of the dialog.

28. In the "Name" column (next to the checkbox) enter "To EOF".

29. In the "Expression" column enter "eof - address".

30. Leave the "Format" empty for now.

31. Click OK and check that the tip is working correctly.

 

Note that because we used the identifier "address" in the expression HexEdit assumes that the result is an address.  And because you did not enter a format specification it displays the address as either hex or decimal depending on how addresses are currently being displayed for the active file.

 

32. Toggle between hex and decimal addresses using the View/Decimal Addresses menu item (or press Alt+D).

33. Check that the display of the tip changes between hex and decimal.

 

When HexEdit displays addresses in hex or decimal it also adjusts the color of the text according to the colors used for hex and decimal addresses in the current color scheme.

 

 

Expressions

 

To make it a bit more interesting, imagine we want to test whether the distance to end of file is divisible by eight.

 

34. Open the Options dialog again.

35. Change the name of the tip from "To EOF" to "EOF dist = QWORD".

36. Change the expression to "(eof - address)%8 == 0".

 

This is expression will be familiar to C/C++ programmers.  It checks if the remainder, when distance to EOF is divided by 8, is zero.

 

Comparing for equality in expressions is done using two equals signs (==).  A single equals sign (=) is used for assignment.  In the above expression this would be an error and no value would be displayed.

 

37. Add a format of "NO".

 

The result of the above expression is a boolean value.  There are only two results for a boolean expression (false or true), but there are many ways to format them, all of which just use the false format as an example (eg: "false", "F", "0", "n", "OFF" etc).

 

 

38. Click OK and verify that when the distance to EOF is 8, 16 etc the tip shows "YES" else it shows "NO".
